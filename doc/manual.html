1 – 简介

Lua 是一种强大、高效、轻量级且可嵌入的脚本语言。 它支持过程式编程、面向对象编程、函数式编程、数据驱动编程以及数据描述。

Lua 将简洁的过程式语法与基于关联数组和可扩展语义的强大数据描述构造相结合。 Lua 是动态类型的，通过基于寄存器的虚拟机解释执行字节码， 并具有自动内存管理及分代垃圾回收机制， 使其非常适合用于配置、脚本编写和快速原型开发。

Lua 以库的形式实现，使用纯 C（即标准 C 和 C++ 的公共子集）编写。 Lua 发行版包含一个名为 lua 的宿主程序， 该程序利用 Lua 库提供了一个完整的、独立的 Lua 解释器， 可用于交互式或批处理操作。 Lua 既可作为强大、轻量级、可嵌入的脚本语言供任何需要它的程序使用， 也可作为独立的、功能强大但轻量高效的语言来使用。

作为一门扩展语言，Lua 没有"主程序"的概念： 它嵌入在宿主客户端中运行， 该宿主客户端被称为嵌入程序或简称为宿主。 （通常，这个宿主就是独立的 lua 程序。） 宿主程序可以调用函数来执行 Lua 代码片段， 可以读写 Lua 变量， 还可以注册供 Lua 代码调用的 C 函数。 通过使用 C 函数，Lua 可以被扩展以处理各种不同的领域， 从而创建出共享语法框架的专用编程语言。

Lua 是自由软件， 按照其许可证的规定，以通常的方式提供且不作任何保证。 本手册所描述的实现可在 Lua 的官方网站 www.lua.org 获取。

与其他参考手册一样， 本文档在某些地方可能较为枯燥。 如需了解 Lua 设计背后的决策讨论， 请参阅 Lua 网站上提供的技术论文。 如需了解 Lua 编程的详细介绍， 请参阅 Roberto 的著作 Programming in Lua（《Lua 编程》）。

2 – 基本概念

本节描述语言的基本概念。

2.1 – 值与类型

Lua 是一种动态类型的语言。 这意味着 变量没有类型；只有值才有类型。 语言中没有类型定义。 所有值都自带其类型。

Lua 中所有的值都是一等值。 这意味着所有值都可以存储在变量中， 作为参数传递给其他函数，以及作为结果返回。

Lua 中有八种基本类型： nil、boolean（布尔）、number（数字）、 string（字符串）、function（函数）、userdata（用户数据）、 thread（线程）和 table（表）。 nil 类型只有一个值，nil， 其主要特性是与其他任何值都不同； 它通常表示没有有用的值。 boolean 类型有两个值，false 和 true。 nil 和 false 都会使条件为假； 它们统称为假值。 任何其他值都会使条件为真。 尽管名为 false， 但它经常被用作 nil 的替代品， 关键区别在于 false 在表中表现得像一个普通值， 而表中的 nil 表示一个不存在的键。

number 类型同时表示 整数和实数（浮点数）， 使用两个子类型：integer（整数）和 float（浮点数）。 标准 Lua 使用 64 位整数和双精度（64 位）浮点数， 但你也可以将 Lua 编译为 使用 32 位整数和/或单精度（32 位）浮点数。 整数和浮点数都使用 32 位的选项 对于小型机器和嵌入式系统特别有吸引力。 （参见文件 luaconf.h 中的宏 LUA_32BITS。）

除非另有说明， 操作整数值时的任何溢出都会回绕， 遵循二进制补码运算的常规规则。 （换句话说， 实际结果是与数学结果在模 2n 意义下相等的唯一可表示整数， 其中 n 是整数类型的位数。）

Lua 对每种子类型何时使用有明确的规则， 但它也会根据需要自动在它们之间进行转换（参见 §3.4.3）。 因此， 程序员可以选择大致忽略整数和浮点数之间的差异， 或者完全控制每个数字的表示方式。

string 类型表示不可变的字节序列。 Lua 是 8 位干净的： 字符串可以包含任何 8 位值， 包括嵌入的零（'\0'）。 Lua 也是与编码无关的； 它不对字符串的内容做任何假设。 Lua 中任何字符串的长度必须能放入一个 Lua 整数中， 字符串加上一个小的头部必须能放入 size_t 中。

Lua 可以调用（和操作）用 Lua 编写的函数以及 用 C 编写的函数（参见 §3.4.10）。 两者都由 function 类型表示。

userdata 类型用于允许将任意 C 数据 存储在 Lua 变量中。 userdata 值表示一块原始内存。 userdata 有两种： 完整 userdata， 即由 Lua 管理的一块内存的对象， 和轻量 userdata， 它只是一个 C 指针值。 Lua 中的 userdata 除了赋值和标识测试外 没有预定义的操作。 通过使用元表， 程序员可以为完整 userdata 值定义操作 （参见 §2.4）。 userdata 值不能在 Lua 中创建或修改， 只能通过 C API 来操作。 这保证了宿主程序和 C 库所拥有数据的完整性。

thread 类型表示独立的执行线程， 用于实现协程（参见 §2.6）。 Lua 线程与操作系统线程无关。 Lua 在所有系统上都支持协程， 即使是那些不原生支持线程的系统。

table 类型实现了关联数组， 即数组的索引不仅可以是数字， 还可以是除 nil 和 NaN 之外的任何 Lua 值。 （NaN（非数字）是 IEEE 754 标准 用于表示未定义数值结果的特殊浮点值， 例如 0/0。） 表可以是异构的； 也就是说，它们可以包含所有类型（除 nil 外）的值。 任何关联到 nil 值的键都不被视为表的一部分。 相反，任何不属于表的键都关联着 nil 值。

表是 Lua 中唯一的数据结构机制； 它们可以用来表示普通数组、列表、 符号表、集合、记录、图、树等。 要表示记录，Lua 使用字段名作为索引。 语言通过提供 a.name 作为 a["name"] 的语法糖来支持这种表示。 在 Lua 中有几种方便的方式来创建表 （参见 §3.4.9）。

与索引一样， 表字段的值可以是任何类型。 特别是， 因为函数是一等值， 表字段可以包含函数。 因此表也可以承载方法（参见 §3.4.11）。

表的索引遵循 语言中原始相等的定义。 表达式 a[i] 和 a[j] 表示相同的表元素 当且仅当 i 和 j 原始相等 （即不使用元方法的相等）。 特别是，具有整数值的浮点数 与其对应的整数相等 （例如，1.0 == 1）。 为了避免歧义， 任何用作键且等于某个整数的浮点数 都会被转换为该整数。 例如，如果你写 a[2.0] = true， 实际插入表中的键将是整数 2。

表、函数、线程和（完整）userdata 值都是对象： 变量实际上并不包含这些值， 而只包含对它们的引用。 赋值、参数传递和函数返回 始终操作对这些值的引用； 这些操作不会产生任何类型的复制。

库函数 type 返回一个描述给定值类型的字符串 （参见 type）。

2.2 – 作用域、变量和环境

变量名引用全局变量还是局部变量 取决于该代码位置上的上下文声明。 （在本讨论中， 函数的形参等同于局部变量。）

所有代码块都以隐式声明 global * 开始， 它将所有自由名称声明为全局变量； 此初始声明在任何其他 global 声明的作用域内都会失效， 如以下示例所示： 

     X = 1       -- Ok, global by default
     do
       global Y  -- voids the implicit initial declaration
       Y = 1     -- Ok, Y declared as global
       X = 1     -- ERROR, X not declared
     end
     X = 2       -- Ok, global by default again

因此，在任何 global 声明之外， Lua 以默认全局的方式工作。 在任何 global 声明内部， Lua 以没有默认值的方式工作： 所有变量都必须声明。

Lua 是一种词法作用域的语言。 变量声明的作用域从声明之后的第一条语句开始， 一直持续到包含该声明的最内层代码块的最后一条非空语句。 （空语句是指标签和空语句。）

一个声明会遮蔽在该声明位置上下文中同名的任何声明。在这个 遮蔽范围内，该名称的任何外部声明都失效。 参见以下示例： 

     global print, x
     x = 10                -- global variable
     do                    -- new block
       local x = x         -- new 'x', with value 10
       print(x)            --> 10
       x = x+1
       do                  -- another block
         local x = x+1     -- another 'x'
         print(x)          --> 12
       end
       print(x)            --> 11
     end
     print(x)              --> 10  (the global one)

注意，在像 local x = x 这样的声明中， 新声明的 x 尚未在作用域内， 因此右边的 x 引用的是外部变量。

由于词法作用域规则， 局部变量可以被在其作用域内定义的函数自由访问。 内部函数使用的局部变量称为上值 （或外部局部变量，或简称外部变量）。

注意，每次执行 local 语句 都会定义新的局部变量。 考虑以下示例： 

     a = {}
     local x = 20
     for i = 1, 10 do
       local y = 0
       a[i] = function () y = y + 1; return x + y end
     end

循环创建了十个闭包 （即匿名函数的十个实例）。 每个闭包使用不同的 y 变量， 而它们都共享同一个 x。

正如我们将在 §3.2 和 §3.3.3 中进一步讨论的， 对全局变量 var 的任何引用 在语法上被转换为 _ENV.var。 此外，每个代码块都在名为 _ENV 的外部局部变量的作用域内编译 （参见 §3.3.2）， 因此 _ENV 本身在代码块中永远不会是一个自由名称。

尽管存在这个外部 _ENV 变量和 自由名称的转换， _ENV 是一个普通的名称。 特别是， 你可以定义以该名称命名的新变量和参数。 （但是，你不应该将 _ENV 定义为全局变量， 否则 _ENV.var 会被转换为 _ENV._ENV.var，如此循环，陷入无限循环。） 对全局变量名的每次引用都使用在该程序位置可见的 _ENV。

用作 _ENV 值的任何表都称为环境。

Lua 维护一个特殊的环境，称为全局环境。 该值存储在 C 注册表中的特殊索引处（参见 §4.3）。 在 Lua 中，全局变量 _G 用相同的值初始化。 （_G 在内部从未被使用， 因此更改其值只会影响你自己的代码。）

当 Lua 加载一个代码块时， 其 _ENV 变量的默认值 是全局环境（参见 load）。 因此，默认情况下， Lua 代码中的全局变量引用全局环境中的条目， 因此它们表现得像传统的全局变量。 此外，所有标准库都在全局环境中加载， 其中的一些函数操作该环境。 你可以使用 load（或 loadfile） 来加载具有不同环境的代码块。 （在 C 中，你必须加载代码块，然后更改 其第一个上值的值；参见 lua_setupvalue。）

2.3 – 错误处理

Lua 中的几个操作可能会引发错误。 错误会中断程序的正常流程， 程序可以通过捕获错误来继续运行。

Lua 代码可以通过调用 error 函数来显式引发错误。 （此函数不会返回。）

要在 Lua 中捕获错误， 你可以进行保护调用， 使用 pcall（或 xpcall）。 函数 pcall 以保护模式调用给定函数。 运行函数时发生的任何错误都会停止其执行， 控制权立即返回给 pcall， 它返回一个状态码。

因为 Lua 是一种嵌入式扩展语言， Lua 代码通过宿主程序中 C 代码的调用来开始运行。 （当你独立使用 Lua 时， lua 应用程序就是宿主程序。） 通常，这个调用是受保护的； 因此，当在编译或执行 Lua 代码块期间 发生未受保护的错误时， 控制权返回给宿主程序， 宿主程序可以采取适当的措施， 例如打印错误消息。

每当发生错误时， 都会传播一个错误对象， 其中包含有关错误的信息。 Lua 本身只生成错误对象为字符串的错误， 但程序可以生成以任何值为错误对象的错误， 除了 nil。 （Lua 会将 nil 错误对象更改为字符串消息。） 由 Lua 程序或其宿主程序来处理此类错误对象。 由于历史原因， 错误对象通常被称为错误消息， 即使它不一定是一个字符串。

当你使用 xpcall（或 C 中的 lua_pcall）时， 你可以提供一个消息处理程序， 在发生错误时被调用。 此函数以原始错误对象作为参数被调用， 并返回一个新的错误对象。 它在错误展开栈之前被调用， 因此它可以收集有关错误的更多信息， 例如检查栈并创建栈回溯。 此消息处理程序仍然受保护调用的保护； 因此，消息处理程序内部的错误 会再次调用消息处理程序。 如果此循环持续时间过长， Lua 会中断它并返回适当的消息。 消息处理程序仅为常规运行时错误调用。 它不会为内存分配错误 或运行终结器或其他消息处理程序时的错误而调用。

Lua 还提供了警告系统（参见 warn）。 与错误不同，警告不会以任何方式干扰 程序执行。 它们通常只向用户生成一条消息， 尽管这种行为可以从 C 中进行调整（参见 lua_setwarnf）。

2.4 – 元表与元方法

Lua 中的每个值都可以有一个元表。 此元表是一个普通的 Lua 表， 它定义了原始值在某些事件下的行为。 你可以通过在元表中设置特定字段来 改变值行为的多个方面。 例如，当非数字值是加法运算的操作数时， Lua 会检查该值元表中 __add 字段的函数。 如果找到该函数， Lua 就调用此函数来执行加法。

元表中每个事件的键是一个以两个下划线开头的字符串， 后跟事件名； 相应的值称为元值。 对于大多数事件，元值必须是一个函数， 该函数随后被称为元方法。 在前面的示例中，键是字符串 "__add"， 元方法是执行加法的函数。 除非另有说明， 元方法实际上可以是任何可调用值， 即函数或具有 __call 元方法的值。

你可以使用 getmetatable 函数 查询任何值的元表。 Lua 使用原始访问来查询元表中的元方法（参见 rawget）。

你可以使用 setmetatable 函数 替换表的元表。 你不能从 Lua 代码中更改其他类型的元表， 除非使用调试库（§6.11）。

表和完整 userdata 有各自的元表， 尽管多个表和 userdata 可以共享它们的元表。 所有其他类型的值每种类型共享一个元表； 也就是说，所有数字共享一个元表， 所有字符串共享一个元表，等等。 默认情况下，值没有元表， 但字符串库为字符串类型设置了一个元表（参见 §6.5）。

下面列出了由元表控制的详细操作列表。 每个事件由其对应的键标识。 按照约定，Lua 使用的所有元表键由 两个下划线后跟小写拉丁字母组成。

  • __add： 加法（+）操作。 如果加法的任何操作数不是数字， Lua 将尝试调用元方法。 它首先检查第一个操作数（即使它是数字）； 如果该操作数没有定义 __add 的元方法， 那么 Lua 将检查第二个操作数。 如果 Lua 能找到元方法， 它将用两个操作数作为参数调用该元方法， 调用的结果 （调整为一个值） 就是操作的结果。 否则，如果未找到元方法， Lua 会引发错误。
  • __sub： 减法（-）操作。 行为类似于加法操作。
  • __mul： 乘法（*）操作。 行为类似于加法操作。
  • __div： 除法（/）操作。 行为类似于加法操作。
  • __mod： 取模（%）操作。 行为类似于加法操作。
  • __pow： 幂运算（^）操作。 行为类似于加法操作。
  • __unm： 取反（一元 -）操作。 行为类似于加法操作。
  • __idiv： 整除（//）操作。 行为类似于加法操作。
  • __band： 按位与（&）操作。 行为类似于加法操作， 但 Lua 会在任何操作数既不是整数 也不是可强制转换为整数的浮点数时尝试元方法（参见 §3.4.3）。
  • __bor： 按位或（|）操作。 行为类似于按位与操作。
  • __bxor： 按位异或（二元 ~）操作。 行为类似于按位与操作。
  • __bnot： 按位非（一元 ~）操作。 行为类似于按位与操作。
  • __shl： 按位左移（<<）操作。 行为类似于按位与操作。
  • __shr： 按位右移（>>）操作。 行为类似于按位与操作。
  • __concat： 连接（..）操作。 行为类似于加法操作， 但 Lua 会在任何操作数既不是字符串也不是数字 （数字总是可以强制转换为字符串）时尝试元方法。
  • __len： 长度（#）操作。 如果对象不是字符串， Lua 将尝试其元方法。 如果有元方法， Lua 以对象作为参数调用它， 调用的结果 （始终调整为一个值） 就是操作的结果。 如果没有元方法但对象是表， 那么 Lua 使用表长度操作（参见 §3.4.7）。 否则，Lua 会引发错误。
  • __eq： 相等（==）操作。 行为类似于加法操作， 但 Lua 只会在被比较的值 都是表或都是完整 userdata 且它们不原始相等时才尝试元方法。 调用的结果始终转换为布尔值。
  • __lt： 小于（<）操作。 行为类似于加法操作， 但 Lua 只会在被比较的值 既不是都是数字也不是都是字符串时才尝试元方法。 此外，调用的结果始终转换为布尔值。
  • __le： 小于等于（<=）操作。 行为类似于小于操作。
  • __index： 索引访问操作 table[key]。 当 table 不是表或者 key 不在 table 中时，会发生此事件。 在 table 的元表中查找元值。

    此事件的元值可以是函数、表或 任何具有 __index 元值的值。 如果是函数， 它以 table 和 key 作为参数被调用， 调用的结果 （调整为一个值） 就是操作的结果。 否则， 最终结果是使用 key 对该元值进行索引的结果。 此索引是常规的，不是原始的， 因此可以触发另一个 __index 元值。

  • __newindex： 索引赋值 table[key] = value。 与索引事件类似， 当 table 不是表或者 key 不在 table 中时，会发生此事件。 在 table 的元表中查找元值。

    与索引类似， 此事件的元值可以是函数、表或 任何具有 __newindex 元值的值。 如果是函数， 它以 table、key 和 value 作为参数被调用。 否则， Lua 在此元值上重复执行索引赋值， 使用相同的键和值。 此赋值是常规的，不是原始的， 因此可以触发另一个 __newindex 元值。

    每当调用 __newindex 元值时， Lua 不执行原始赋值。 如果需要， 元方法本身可以调用 rawset 来执行赋值。

  • __call： 调用操作 func(args)。 当 Lua 尝试调用一个非函数值 （即 func 不是函数）时，会发生此事件。 在 func 中查找元方法。 如果存在， 元方法以 func 作为其第一个参数被调用， 后面跟着原始调用的参数（args）。 调用的所有结果 就是操作的结果。 这是唯一允许多个结果的元方法。

除了上面的列表外， 解释器还遵守元表中的以下键： __gc（参见 §2.5.3）、 __close（参见 §3.3.8）、 __mode（参见 §2.5.4） 和 __name。 （__name 条目， 当它包含一个字符串时， 可能被 tostring 和错误消息使用。）

对于一元运算符（取反、长度和按位非）， 元方法被计算并以一个等于第一个操作数的虚拟第二个操作数来调用。 这个额外的操作数只是为了简化 Lua 的内部实现 （使这些运算符表现得像二元运算）， 在未来的版本中可能会被移除。 对于大多数用途来说，这个额外的操作数无关紧要。

因为元表是普通的表， 它们可以包含任意字段， 而不仅仅是上面定义的事件名称。 标准库中的一些函数 （例如 tostring） 将元表中的其他字段用于自己的目的。

在将表设置为某个对象的元表之前， 最好将所有需要的元方法添加到该表中。 特别是，__gc 元方法只有在遵循此顺序时才有效 （参见 §2.5.3）。 最好在对象创建后立即设置其元表。

2.5 – 垃圾回收

Lua 执行自动内存管理。 这意味着 你不必为新对象分配内存， 也不必在对象不再需要时释放它。 Lua 通过运行 垃圾收集器来自动管理内存，收集所有已死亡的对象。 Lua 使用的所有内存都受自动管理： 字符串、表、userdata、函数、线程、内部结构等。

一旦收集器能够确定对象 在程序正常执行中不再被访问，该对象就被视为已死亡。 （"正常执行"这里不包括终结器， 终结器会复活已死亡的对象（参见 §2.5.3）， 也不包括使用调试库的一些操作。） 注意，收集器能够确定对象已死亡的时间 可能与程序员的预期不一致。 唯一的保证是 Lua 不会收集 在程序正常执行中仍可能被访问的对象， 并且最终会收集 从 Lua 无法访问的对象。 （这里， 从 Lua 无法访问意味着既没有变量也没有 其他存活对象引用该对象。） 因为 Lua 不了解 C 代码， 它永远不会收集通过注册表可访问的对象（参见 §4.3）， 其中包括全局环境（参见 §2.2）和 主线程。

Lua 中的垃圾收集器（GC）可以在两种模式下工作： 增量模式和分代模式。

使用默认参数的默认 GC 模式 对大多数用途来说已经足够。 但是，将大量时间用于分配和释放内存的程序 可以从其他设置中受益。 请记住，GC 行为在不同平台和 不同 Lua 版本之间是不可移植的； 因此，最优设置也是不可移植的。

你可以通过在 C 中调用 lua_gc 或在 Lua 中调用 collectgarbage 来更改 GC 模式和参数。 你也可以使用这些函数直接控制收集器， 例如停止或重启它。

2.5.1 – 增量垃圾回收

在增量模式下， 每个 GC 周期以小步骤执行标记-清除收集， 与程序执行交错进行。 在此模式下， 收集器使用三个数字来控制其垃圾回收周期： 垃圾收集器暂停、 垃圾收集器步进乘数 和 垃圾收集器步进大小。

垃圾收集器暂停 控制收集器在开始新周期之前等待的时间。 当字节数达到上次收集后总字节数的 n% 时， 收集器开始新周期。 较大的值使收集器不那么激进。 等于或小于 100 的值意味着收集器不会等待 就开始新周期。 值为 200 意味着收集器等待 总字节数翻倍后才开始新周期。

垃圾收集器步进大小控制 每个增量步骤的大小， 特别是解释器在执行一步之前分配多少字节： 值为 n 意味着解释器将在步骤之间分配 大约 n 个字节。

垃圾收集器步进乘数 控制每个增量步骤执行多少工作。 值为 n 意味着解释器将为每个分配的字节执行 n% 个工作单元。 一个工作单元大致对应于遍历一个槽 或清除一个对象。 较大的值使收集器更激进。 请注意，太小的值可能使收集器 太慢以至于永远无法完成一个周期。 作为特殊情况，零值意味着无限的工作， 实际上产生了一个非增量的、停止世界的收集器。

2.5.2 – 分代垃圾回收

在分代模式下， 收集器执行频繁的小收集， 仅遍历最近创建的对象。 如果小收集后字节数超过某个限制， 收集器会切换到大收集， 遍历所有对象。 然后收集器将继续进行大收集， 直到它检测到程序产生了足够多的垃圾， 足以证明切回小收集是合理的。

分代模式使用三个参数： 小收集乘数、小到大乘数 和 大到小乘数。

小收集乘数控制小收集的频率。 对于小收集乘数 x， 当字节数比上次大收集后使用的字节数 增长 x% 时，将执行新的小收集。 例如，对于乘数 20， 当字节数比上次大收集后的总数 增长 20% 时，收集器将执行小收集。

小到大乘数控制切换到大收集的时机。 对于乘数 x， 当老对象的字节数比上次大收集后的总数 增长 x% 时， 收集器将切换到大收集。 例如，对于乘数 100， 当老字节数比上次大收集后的总数 增长两倍时，收集器将执行大收集。 作为特殊情况， 值为 0 会阻止收集器执行大收集。

大到小乘数控制切回小收集的时机。 对于乘数 x， 当一次大收集收集了上一个周期中分配的字节数的至少 x% 时， 收集器将切回小收集。 特别是，对于乘数为 0， 收集器将在执行一次大收集后立即切回小收集。

2.5.3 – 垃圾回收元方法

你可以为表设置垃圾回收元方法， 并且使用 C API， 也可以为完整 userdata 设置（参见 §2.4）。 这些元方法称为终结器， 当垃圾收集器检测到相应的表或 userdata 已死亡时被调用。 终结器允许你将 Lua 的垃圾回收与外部资源管理协调起来， 例如关闭文件、网络或数据库连接， 或释放你自己的内存。

要使对象（表或 userdata）在被收集时被终结， 你必须将其标记为待终结。 当你设置元表且元表有 __gc 元方法时， 你就将对象标记为待终结。 注意，如果你设置了一个没有 __gc 字段的元表， 然后在元表中创建该字段， 该对象不会被标记为待终结。

当一个被标记的对象变为已死亡时， 它不会立即被垃圾收集器收集。 相反，Lua 将其放入一个列表中。 收集之后， Lua 会遍历该列表。 对于列表中的每个对象， 它检查对象的 __gc 元方法： 如果存在， Lua 以该对象作为其唯一参数调用它。

在每个垃圾回收周期结束时， 终结器按照对象被标记为待终结的相反顺序被调用， 在该周期收集的对象中； 也就是说，第一个被调用的终结器是 与程序中最后标记的对象关联的那个。 每个终结器的执行可能发生在常规代码执行期间的任何时候。

因为被收集的对象仍然必须被终结器使用， 该对象（以及仅通过它可访问的其他对象） 必须被 Lua 复活。 通常，这种复活是暂时的， 对象内存将在下一个垃圾回收周期中被释放。 然而，如果终结器将对象存储在某个全局位置 （例如全局变量）， 那么复活就是永久的。 此外，如果终结器再次将待终结对象标记为待终结， 其终结器将在对象再次死亡的下一个周期中被再次调用。 在任何情况下， 对象内存只在对象已死亡且未被标记为待终结的 GC 周期中被释放。

当你关闭状态（参见 lua_close）时， Lua 按照标记的相反顺序调用所有标记为待终结对象的终结器。 如果任何终结器在此阶段将对象标记为待收集， 这些标记不会产生任何效果。

终结器不能让出也不能运行垃圾收集器。 因为它们可能在不可预测的时间运行， 最好将每个终结器限制在正确释放 其关联资源所需的最少操作。

运行终结器时发生的任何错误都会生成警告； 错误不会被传播。

2.5.4 – 弱表

弱表是其元素为 弱引用的表。 弱引用被垃圾收集器忽略。 换句话说， 如果对一个对象的唯一引用是弱引用， 那么垃圾收集器将收集该对象。

弱表可以有弱键、弱值或两者兼有。 具有弱值的表允许收集其值， 但阻止收集其键。 同时具有弱键和弱值的表允许 收集键和值。 在任何情况下，如果键或值被收集， 该键值对就会从表中移除。 表的弱性由其元表的 __mode 字段控制。 此元值（如果存在）必须是以下字符串之一： "k"，表示具有弱键的表； "v"，表示具有弱值的表； 或 "kv"，表示同时具有弱键和弱值的表。

具有弱键和强值的表 也称为临时表（ephemeron table）。 在临时表中， 只有当其键可到达时，值才被视为可到达。 特别是， 如果对键的唯一引用来自其值， 则该键值对会被移除。

表弱性的任何更改可能仅在下一个收集周期时生效。 特别是，如果你将弱性更改为更强的模式， Lua 可能在更改生效之前仍然收集该表中的一些项。

只有具有显式构造的对象才会从弱表中移除。 值（如数字和轻量 C 函数） 不受垃圾回收影响， 因此不会从弱表中移除 （除非其关联的值被收集）。 虽然字符串受垃圾回收影响， 但它们没有显式构造且 其相等性是按值比较的； 它们更像是值而不是对象。 因此，它们不会从弱表中移除。

复活的对象 （即正在被终结的对象 以及仅通过正在被终结的对象可访问的对象） 在弱表中有特殊的行为。 它们在运行终结器之前从弱值中移除， 但在运行终结器之后的下一次收集中 才从弱键中移除，此时这些对象才被真正释放。 此行为允许终结器通过弱表访问与对象关联的属性。

如果弱表是一个收集周期中被复活的对象之一， 它可能要到下一个周期才能被正确清除。

2.6 – 协程

Lua 支持协程， 也称为协作式多线程。 Lua 中的协程表示一个独立的执行线程。 但是，与多线程系统中的线程不同， 协程只能通过显式调用让出函数来挂起其执行。

你通过调用 coroutine.create 来创建协程。 它的唯一参数是一个函数， 该函数是协程的主函数。 create 函数只创建一个新协程并 返回它的句柄（一个 thread 类型的对象）； 它不会启动协程。

你通过调用 coroutine.resume 来执行协程。 当你第一次调用 coroutine.resume， 将 coroutine.create 返回的线程 作为其第一个参数传递时， 协程通过调用其主函数开始执行。 传递给 coroutine.resume 的额外参数 作为该函数的参数传递。 协程开始运行后， 它会一直运行直到终止或让出。

协程可以通过两种方式终止其执行： 正常终止，当其主函数返回时 （显式或隐式，在最后一条指令之后）； 以及异常终止，当发生未保护的错误时。 在正常终止的情况下， coroutine.resume 返回 true， 加上协程主函数返回的任何值。 在错误的情况下，coroutine.resume 返回 false 加上错误对象。 在这种情况下，协程不会展开其栈， 因此可以在错误后使用调试 API 来检查它。

协程通过调用 coroutine.yield 来让出。 当协程让出时， 相应的 coroutine.resume 立即返回， 即使让出发生在嵌套函数调用内部 （即不在主函数中， 而是在主函数直接或间接调用的函数中）。 在让出的情况下，coroutine.resume 也返回 true， 加上传递给 coroutine.yield 的任何值。 下次你恢复同一个协程时， 它从让出的点继续执行， coroutine.yield 的调用返回传递给 coroutine.resume 的额外参数。

与 coroutine.create 类似， coroutine.wrap 函数也创建一个协程， 但它不是返回协程本身， 而是返回一个函数，当被调用时恢复协程。 传递给此函数的任何参数 作为额外参数传递给 coroutine.resume。 coroutine.wrap 返回 coroutine.resume 返回的所有值， 除了第一个（布尔错误代码）。 与 coroutine.resume 不同， coroutine.wrap 创建的函数 会将任何错误传播给调用者。 在这种情况下， 该函数也会关闭协程（参见 coroutine.close）。

作为协程如何工作的示例， 考虑以下代码： 

     function foo (a)
       print("foo", a)
       return coroutine.yield(2*a)
     end
     
     co = coroutine.create(function (a,b)
           print("co-body", a, b)
           local r = foo(a+1)
           print("co-body", r)
           local r, s = coroutine.yield(a+b, a-b)
           print("co-body", r, s)
           return b, "end"
     end)
     
     print("main", coroutine.resume(co, 1, 10))
     print("main", coroutine.resume(co, "r"))
     print("main", coroutine.resume(co, "x", "y"))
     print("main", coroutine.resume(co, "x", "y"))

当你运行它时，它会产生以下输出： 

     co-body 1       10
     foo     2
     main    true    4
     co-body r
     main    true    11      -9
     co-body x       y
     main    true    10      end
     main    false   cannot resume dead coroutine

你也可以通过 C API 创建和操作协程： 参见函数 lua_newthread、lua_resume 和 lua_yield。

3 – 语言

本节描述 Lua 的词法、语法和语义。 换句话说， 本节描述 哪些词法单元是合法的， 它们如何组合， 以及这些组合的含义。

语言结构将使用常见的扩展 BNF 表示法来解释， 其中 {a} 表示 0 个或多个 a， [a] 表示可选的 a。 非终结符以非终结符的形式显示， 关键字以 kword 的形式显示， 其他终结符号以 ‘=’ 的形式显示。 Lua 的完整语法可在本手册末尾的 §9 中找到。

3.1 – 词法约定

Lua 是一种自由格式的语言。 它忽略词法元素（token）之间的空格和注释， 仅将它们作为两个 token 之间的分隔符。 在源代码中， Lua 将标准 ASCII 空白字符视为空格，包括空格、换页符、换行符、 回车符、水平制表符和垂直制表符。

名称 （也称为标识符） 可以是任何由拉丁字母、 阿拉伯-印度数字和下划线组成的字符串， 但不能以数字开头， 也不能是保留字。 标识符用于命名变量、表字段和标签。

以下关键字是保留的， 不能用作名称： 

     and       break     do        else      elseif    end
     false     for       function  global    goto      if
     in        local     nil       not       or        repeat
     return    then      true      until     while

Lua 是大小写敏感的语言： and 是保留字，但 And 和 AND 是两个不同的合法名称。 按照惯例， 程序应避免创建以下划线开头后跟一个或多个大写字母的名称 （例如 _VERSION）。

以下字符串表示其他 token： 

     +     -     *     /     %     ^     #
     &     ~     |     <<    >>    //
     ==    ~=    <=    >=    <     >     =
     (     )     {     }     [     ]     ::
     ;     :     ,     .     ..    ...

短字面字符串 可以由匹配的单引号或双引号界定， 并且可以包含以下类 C 转义序列： '\a'（响铃）、 '\b'（退格）、 '\f'（换页）、 '\n'（换行）、 '\r'（回车）、 '\t'（水平制表符）、 '\v'（垂直制表符）、 '\\'（反斜杠）、 '\"'（引号 [双引号]）以及 '\''（撇号 [单引号]）。 反斜杠后跟换行符会在字符串中产生一个换行。 转义序列 '\z' 会跳过其后的空白字符 （包括换行符）； 这对于将长字面字符串拆分和缩进到多行特别有用， 而不会将换行符和空格添加到字符串内容中。 短字面字符串不能包含未转义的换行符， 也不能包含未形成有效转义序列的转义。

我们可以通过数值指定短字面字符串中的任何字节， 包括嵌入的零值。 这可以通过转义序列 \xXX （其中 XX 是恰好两个十六进制数字的序列）来实现， 或者通过转义序列 \ddd （其中 ddd 是最多三个十进制数字的序列）来实现。 （请注意，如果十进制转义序列后要跟一个数字， 则必须使用恰好三个数字来表示。）

Unicode 字符的 UTF-8 编码 可以通过转义序列 \u{XXX} （必须使用花括号括起来）插入到字面字符串中， 其中 XXX 是一个或多个十六进制数字的序列， 表示字符的码位。 此码位可以是任何小于 231 的值。 （Lua 在这里使用原始的 UTF-8 规范， 不受有效 Unicode 码位的限制。）

字面字符串也可以使用由长方括号括起来的长格式定义。 我们将级别 n 的左长方括号定义为一个左方括号 后跟 n 个等号再跟另一个左方括号。 因此，级别为 0 的左长方括号写作 [[， 级别为 1 的左长方括号写作 [=[， 以此类推。 右长方括号的定义类似； 例如， 级别为 4 的右长方括号写作 ]====]。 长字面量以任何级别的左长方括号开始， 以相同级别的第一个右长方括号结束。 它可以包含除相同级别的右方括号之外的任何文本。 这种括号形式的字面量可以跨越多行， 不解释任何转义序列， 并忽略任何其他级别的长方括号。 任何类型的行尾序列 （回车、换行、回车后跟换行、或换行后跟回车） 都会被转换为简单的换行符。 当左长方括号后紧跟换行符时， 该换行符不会包含在字符串中。

例如，在使用 ASCII 的系统中 （其中 'a' 编码为 97， 换行符编码为 10，'1' 编码为 49）， 以下五个字面字符串表示相同的字符串： 

     a = 'alo\n123"'
     a = "alo\n123\""
     a = '\97lo\10\04923"'
     a = [[alo
     123"]]
     a = [==[
     alo
     123"]==]

字面字符串中未被上述规则明确影响的任何字节都代表其自身。 但是，Lua 以文本模式打开文件进行解析， 系统的文件函数可能对某些控制字符存在问题。 因此，更安全的做法是将二进制数据 表示为带有显式转义序列的引号字面量， 对非文本字符使用转义序列。

数字常量（或数值） 可以带有可选的小数部分和可选的十进制指数来书写， 由字母 'e' 或 'E' 标记。 Lua 也接受十六进制常量， 以 0x 或 0X 开头。 十六进制常量还接受可选的小数部分 加上可选的二进制指数， 由字母 'p' 或 'P' 标记并以十进制书写。 （例如，0x1.fp10 表示 1984， 即 0x1f / 16 乘以 210。）

带有小数点或指数的数字常量表示浮点数； 否则， 如果其值适合整数或者是十六进制常量， 则表示整数； 否则（即溢出的十进制整数），表示浮点数。 既没有小数点也没有指数的十六进制数值 始终表示整数值； 如果值溢出，它会回绕以适应有效的整数范围。

有效整数常量的示例有 

     3   345   0xff   0xBEBADA

有效浮点常量的示例有 

     3.0     3.1416     314.16e-2     0.31416E1     34e1
     0x0.1E  0xA23p-4   0X1.921FB54442D18P+1

注释在字符串外部的任何位置以双连字符（--）开始。 如果 -- 紧跟的文本不是左长方括号， 则该注释为短注释， 持续到行尾。 否则，它就是长注释， 持续到对应的右长方括号。

3.2 – 变量

变量是存储值的地方。 Lua 中有三种变量： 全局变量、局部变量和表字段。

单个名称可以表示全局变量或局部变量 （或函数的形式参数， 它是局部变量的一种特殊形式）（参见 §2.2）： 

	var ::= Name

Name 表示标识符（参见 §3.1）。

由于变量是词法作用域的， 定义在其作用域内的函数可以自由访问局部变量 （参见 §2.2）。

在对变量进行第一次赋值之前，其值为 nil。

方括号用于索引表： 

	var ::= prefixexp ‘[’ exp ‘]’

对表字段的访问含义可以通过元表来改变 （参见 §2.4）。

语法 var.Name 只是 var["Name"] 的语法糖： 

	var ::= prefixexp ‘.’ Name

对全局变量 x 的访问 等价于 _ENV.x。

3.3 – 语句

Lua 支持几乎常规的语句集， 类似于其他常规语言中的语句。 该集合包括 块、赋值、控制结构、函数调用 和变量声明。

3.3.1 – 块

块是按顺序执行的语句列表： 

	block ::= {stat}

Lua 有空语句， 允许你用分号分隔语句， 以分号开始一个块， 或连续写两个分号： 

	stat ::= ‘;’

函数调用和赋值都可以以左括号开头。 这种可能性会导致 Lua 语法的歧义。 考虑以下代码片段： 

     a = b + c
     (print or io.write)('done')

语法可以以两种方式理解这段代码： 

     a = b + c(print or io.write)('done')
     
     a = b + c; (print or io.write)('done')

当前的解析器总是以第一种方式理解这种构造， 将左括号解释为调用参数的开始。 为了避免这种歧义， 最好在以括号开头的语句前加上分号： 

     ;(print or io.write)('done')

一个块可以被显式界定以产生单个语句： 

	stat ::= do block end

显式块对于控制变量声明的作用域很有用。 显式块有时也用于在另一个块的中间 添加 return 语句（参见 §3.3.4）。

3.3.2 – 代码块

Lua 的编译单元称为代码块。 从语法上讲， 代码块就是一个块： 

	chunk ::= block

Lua 将代码块作为具有可变数量参数的匿名函数体来处理 （参见 §3.4.11）。 因此，代码块可以定义局部变量、 接收参数和返回值。 此外，这种匿名函数的编译是在名为 _ENV 的外部局部变量的作用域中进行的 （参见 §2.2）。 所得的函数始终将 _ENV 作为其唯一的外部变量， 即使它没有使用该变量。

代码块可以存储在文件中或宿主程序内的字符串中。 要执行代码块， Lua 首先加载它， 将代码块的代码预编译为虚拟机的指令， 然后 Lua 使用虚拟机的解释器执行编译后的代码。

代码块也可以被预编译为二进制形式； 详情请参见 luac 程序和 string.dump 函数。 源代码和编译形式的程序是可互换的； Lua 会自动检测文件类型并相应地处理（参见 load）。 请注意，与源代码不同， 恶意构造的二进制代码块可能会导致解释器崩溃。

3.3.3 – 赋值

Lua 允许多重赋值。 因此，赋值的语法 定义了左侧的变量列表 和右侧的表达式列表。 两个列表中的元素用逗号分隔： 

	stat ::= varlist ‘=’ explist
	varlist ::= var {‘,’ var}
	explist ::= exp {‘,’ exp}

表达式在 §3.4 中讨论。

在赋值之前， 值列表会根据变量列表的长度进行调整 （参见 §3.4.12）。

如果一个变量在多重赋值中既被赋值又被读取， Lua 确保所有读取操作都获得赋值前的变量值。 因此以下代码 

     i = 3
     i, a[i] = i+1, 20

将 a[3] 设为 20，而不影响 a[4]， 因为 a[i] 中的 i 在被赋值为 4 之前已被求值为 3。 类似地，以下代码 

     x, y = y, x

交换 x 和 y 的值， 而 

     x, y, z = y, z, x

循环置换 x、y 和 z 的值。

请注意，此保证仅涵盖语法上在赋值语句内的访问。 如果在赋值期间调用的函数或元方法改变了变量的值， Lua 不保证该访问的顺序。

对全局名称 x = val 的赋值 等价于赋值 _ENV.x = val（参见 §2.2）。

对表字段和全局变量（它们实际上也是表字段）的赋值含义 可以通过元表来改变（参见 §2.4）。

3.3.4 – 控制结构

控制结构 if、while 和 repeat 具有常见的含义和熟悉的语法： 

	stat ::= while exp do block end
	stat ::= repeat block until exp
	stat ::= if exp then block {elseif exp then block} [else block] end

Lua 还有一个 for 语句，有两种形式（参见 §3.3.5）。

控制结构的条件表达式可以返回任何值。 false 和 nil 的求值结果为假。 所有不同于 nil 和 false 的值求值结果为真。 特别地，数字 0 和空字符串也求值为真。

在 repeat–until 循环中， 内部块不在 until 关键字处结束， 而只在条件之后结束。 因此，条件可以引用在循环块内声明的局部变量。

goto 语句将程序控制权转移到标签。 由于语法原因， Lua 中的标签也被认为是语句： 

	stat ::= goto Name
	stat ::= label
	label ::= ‘::’ Name ‘::’

标签在其定义的整个块中可见， 嵌套函数内部除外。 goto 可以跳转到任何可见的标签，只要它不会 进入变量声明的作用域。 不应在先前同名标签可见的地方声明标签， 即使该其他标签已在外部块中声明。

break 语句终止 while、repeat 或 for 循环的执行， 跳转到循环之后的下一条语句： 

	stat ::= break

break 结束最内层的封闭循环。

return 语句用于从函数或代码块 （它被当作匿名函数来处理）返回值。 函数可以返回多个值， 因此 return 语句的语法是 

	stat ::= return [explist] [‘;’]

return 语句只能作为块的最后一条语句书写。 如果需要在块的中间 return， 则可以使用显式内部块， 如惯用写法 do return end， 因为现在 return 是其（内部）块的最后一条语句。

3.3.5 – For 语句

for 语句有两种形式： 数值型和泛型。

数值型 for 循环

数值型 for 循环在控制变量遍历等差数列时重复执行一段代码。 其语法如下： 

	stat ::= for Name ‘=’ exp ‘,’ exp [‘,’ exp] do block end

给定的标识符（Name）定义控制变量， 它是循环体（块）内的一个新的只读（const）局部变量。

循环首先对三个控制表达式求值一次。 它们的值分别称为 初始值、上限和步长。 如果步长缺失，则默认为 1。

如果初始值和步长都是整数， 则循环以整数方式执行； 注意上限可能不是整数。 否则，三个值将被转换为浮点数， 循环以浮点数方式执行。 在此情况下请注意浮点精度。

在初始化之后， 循环体在控制变量的值遍历等差数列时重复执行， 从初始值开始， 公差由步长给出。 负步长产生递减序列； 步长等于零会引发错误。 循环在值小于或等于上限时继续 （对于负步长则是大于或等于）。 如果初始值已经大于上限 （或小于，如果步长为负）， 则不会执行循环体。

对于整数循环， 控制变量永远不会回绕； 相反，循环在溢出时结束。

泛型 for 循环

泛型 for 语句在称为迭代器的函数上工作。 在每次迭代中，调用迭代器函数以产生一个新值， 当此新值为 nil 时停止。 泛型 for 循环具有以下语法： 

	stat ::= for namelist in explist do block end
	namelist ::= Name {‘,’ Name}

像这样的 for 语句 

     for var_1, ···, var_n in explist do body end

的工作方式如下。

名称 var_i 声明了循环体内的局部循环变量。 这些变量中的第一个是控制变量， 它是一个只读（const）变量。

循环首先对 explist 求值以产生四个值： 一个迭代器函数、 一个状态、 控制变量的初始值 和一个关闭值。

然后，在每次迭代中， Lua 用两个参数调用迭代器函数： 状态和控制变量。 此调用的结果随后被赋给循环变量， 遵循多重赋值的规则（参见 §3.3.3）。 如果控制变量变为 nil， 循环终止。 否则，执行循环体，循环进入下一次迭代。

关闭值的行为类似于 待关闭变量（参见 §3.3.8）， 可用于在循环结束时释放资源。 否则，它不会干扰循环。

3.3.6 – 函数调用作为语句

为了允许可能的副作用， 函数调用可以作为语句执行： 

	stat ::= functioncall

在这种情况下，所有返回值都会被丢弃。 函数调用在 §3.4.10 中解释。

3.3.7 – 变量声明

局部变量和全局变量可以在块内的任何位置声明。 声明可以包括初始化： 

	stat ::= local attnamelist [‘=’ explist]
	stat ::= global attnamelist [‘=’ explist]

如果没有初始化， 局部变量被初始化为 nil； 全局变量保持不变。 否则，初始化将获得与多重赋值相同的调整 （参见 §3.3.3）。 此外，对于全局变量， 如果变量已经定义（即具有非 nil 值）， 初始化将引发运行时错误。

名称列表可以前缀一个属性 （尖括号之间的名称）， 每个变量名可以后缀一个属性： 

	attnamelist ::=  [attrib] Name [attrib] {‘,’ Name [attrib]}
	attrib ::= ‘<’ Name ‘>’

前缀属性应用于列表中的所有名称； 后缀属性应用于其特定名称。 有两种可能的属性： const，声明常量或只读变量， 即不能用作赋值左侧的变量， 以及 close，声明待关闭变量（参见 §3.3.8）。 只有局部变量可以具有 close 属性。 变量列表最多可以包含一个待关闭变量。

Lua 还提供全局变量的集体声明： 

	stat ::= global [attrib] ‘*’

此特殊形式隐式地将之前未显式声明的所有名称声明为全局变量。 特别地， global<const> * 将之前未显式声明的所有名称 隐式声明为只读全局变量； 参见以下示例： 

     global X
     global<const> *
     print(math.pi)   -- Ok, 'print' and 'math' are read-only
     X = 1            -- Ok, declared as read-write
     Y = 1            -- Error, Y is read-only

如 §2.2 中所述， 所有代码块都以隐式声明 global * 开始， 但此前导声明在任何其他 global 声明的作用域内会失效。 因此，不使用全局声明或以 global * 开头的程序 可以自由地读写访问任何全局变量； 以 global<const> * 开头的程序 可以自由地只读访问任何全局变量； 而以任何其他全局声明开头的程序 （例如 global none）只能引用已声明的变量。

请注意，对于全局变量， 任何声明的效果仅在语法上 （可选赋值除外）： 

     global X <const>, _G
     X = 1           -- ERROR
     _ENV.X = 1      -- Ok
     _G.print(X)     -- Ok
     foo()         -- 'foo' can freely change any global

代码块也是一个块（参见 §3.3.2）， 因此变量可以在代码块中任何显式块之外声明。

变量声明的可见性规则 在 §2.2 中解释。

3.3.8 – 待关闭变量

待关闭变量的行为类似于常量局部变量， 不同之处在于，当变量超出作用域时， 其值会被关闭， 包括正常块终止、 通过 break/goto/return 退出其块， 或因错误退出。

这里，关闭一个值意味着 调用其 __close 元方法。 调用元方法时， 值本身作为第一个参数传递。 如果发生了错误， 导致退出的错误对象 作为第二个参数传递； 否则，没有第二个参数。

赋给待关闭变量的值 必须具有 __close 元方法 或为假值。 （nil 和 false 作为待关闭值时会被忽略。）

如果多个待关闭变量在同一事件中超出作用域， 它们按声明的相反顺序关闭。

如果在运行关闭方法时发生任何错误， 该错误的处理方式与变量定义所在常规代码中的错误相同。 在错误之后， 其他待处理的关闭方法仍将被调用。

如果一个协程让出执行权后永远不再恢复， 某些变量可能永远不会超出作用域， 因此永远不会被关闭。 （这些变量是在协程内部创建的 并且在协程让出时在其作用域内的变量。） 类似地，如果协程以错误结束， 它不会展开其栈， 因此不会关闭任何变量。 在两种情况下， 你可以使用终结器 或调用 coroutine.close 来关闭变量。 但是，如果协程是通过 coroutine.wrap 创建的， 则在发生错误时，其对应的函数将关闭协程。

3.4 – 表达式

Lua 中的基本表达式如下： 

	exp ::= prefixexp
	exp ::= nil | false | true
	exp ::= Numeral
	exp ::= LiteralString
	exp ::= functiondef
	exp ::= tableconstructor
	exp ::= ‘...’
	exp ::= exp binop exp
	exp ::= unop exp
	prefixexp ::= var | functioncall | ‘(’ exp ‘)’

数值和字面字符串在 §3.1 中解释； 变量在 §3.2 中解释； 函数定义在 §3.4.11 中解释； 函数调用在 §3.4.10 中解释； 表构造器在 §3.4.9 中解释。 可变参数表达式 由三个点（'...'）表示，只能在 可变参数函数内部直接使用； 它们在 §3.4.11 中解释。

二元运算符包括算术运算符（参见 §3.4.1）、 位运算符（参见 §3.4.2）、 关系运算符（参见 §3.4.4）、逻辑运算符（参见 §3.4.5）， 以及连接运算符（参见 §3.4.6）。 一元运算符包括一元减号（参见 §3.4.1）、 一元位运算 NOT（参见 §3.4.2）、 一元逻辑 not（参见 §3.4.5）， 以及一元长度运算符（参见 §3.4.7）。

3.4.1 – 算术运算符

Lua 支持以下算术运算符：

  • +：加法
  • -：减法
  • *：乘法
  • /：浮点除法
  • //：向下取整除法
  • %：取模
  • ^：幂运算
  • -：一元减号

除了幂运算和浮点除法之外， 算术运算符的工作方式如下： 如果两个操作数都是整数， 则在整数上执行操作，结果为整数。 否则，如果两个操作数都是数字， 则将它们转换为浮点数， 按照机器的浮点算术规则执行操作 （通常是 IEEE 754 标准）， 结果为浮点数。 （字符串库在算术运算中会将字符串强制转换为数字； 详情请参见 §3.4.3。）

幂运算和浮点除法（/） 始终将其操作数转换为浮点数， 结果始终为浮点数。 幂运算使用 ISO C 函数 pow， 因此它也适用于非整数指数。

向下取整除法（//）是一种将商向负无穷方向舍入的除法， 结果为其操作数除法的向下取整值。

取模定义为将商向负无穷方向舍入的除法（向下取整除法）的余数。

在整数算术溢出的情况下， 所有操作都会回绕。

3.4.2 – 位运算符

Lua 支持以下位运算符：

  • &：按位与
  • |：按位或
  • ~：按位异或
  • >>：右移
  • <<：左移
  • ~：一元按位非

所有位运算将其操作数转换为整数 （参见 §3.4.3）， 操作这些整数的所有位， 结果为整数。

右移和左移都用零填充空位。 负位移向相反方向移动； 绝对值等于或大于整数位数的位移结果为零（因为所有位都被移出）。

3.4.3 – 强制转换和类型转换

Lua 在运行时提供某些类型和表示之间的自动转换。 位运算符始终将浮点操作数转换为整数。 幂运算和浮点除法 始终将整数操作数转换为浮点数。 应用于混合数字（整数和浮点数）的所有其他算术操作 将整数操作数转换为浮点数。 C API 也会根据需要将整数转换为浮点数， 将浮点数转换为整数。 此外，字符串连接除了字符串外还接受数字作为参数。

在从整数到浮点数的转换中， 如果整数值有精确的浮点表示， 那就是结果。 否则， 转换取最近的较高或较低的可表示值。 这种转换永远不会失败。

从浮点数到整数的转换 检查浮点数是否具有精确的整数表示 （即浮点数具有整数值并且在整数表示范围内）。 如果是，则该表示就是结果。 否则，转换失败。

Lua 中有多个地方在必要时将字符串强制转换为数字。 特别地， 字符串库设置元方法来尝试在所有算术运算中 将字符串强制转换为数字。 如果转换失败， 库会调用另一个操作数的元方法 （如果存在）或引发错误。 请注意，位运算符不会进行此强制转换。

不依赖从字符串到数字的隐式强制转换总是一个好习惯， 因为它们并不总是被应用； 特别地，"1"==1 为假，"1"<1 会引发错误 （参见 §3.4.4）。 这些强制转换主要为了兼容性而存在， 可能会在该语言的未来版本中被移除。

字符串根据其语法和 Lua 词法分析器的规则 被转换为整数或浮点数。 字符串还可以有前导和尾随空白以及符号。 所有从字符串到数字的转换 都接受点和当前区域设置标记作为基数字符。 （但是，Lua 词法分析器只接受点。） 如果字符串不是有效的数字， 转换失败。 如有必要，此第一步的结果随后根据先前浮点数和整数之间转换的规则 转换为特定的数字子类型。

从数字到字符串的转换使用非指定的可读格式。 要以任何特定方式将数字转换为字符串， 请使用 string.format 函数。

3.4.4 – 关系运算符

Lua 支持以下关系运算符：

  • ==：等于
  • ~=：不等于
  • <：小于
  • >：大于
  • <=：小于等于
  • >=：大于等于

这些运算符始终返回 false 或 true。

等于（==）首先比较其操作数的类型。 如果类型不同，则结果为 false。 否则，比较操作数的值。 如果字符串具有相同的字节内容，则它们相等。 如果数字表示相同的数学值，则它们相等。

表、userdata 和线程 按引用比较： 只有当两个对象是同一个对象时，它们才被认为是相等的。 每次创建新对象 （表、userdata 或线程）时， 此新对象都与任何先前存在的对象不同。 函数始终与自身相等。 具有任何可检测差异的函数 （不同的行为、不同的定义）始终是不同的。 在不同时间创建但没有可检测差异的函数 可能被归类为相等或不相等 （取决于内部缓存细节）。

你可以通过使用 __eq 元方法来改变 Lua 比较表和 userdata 的方式 （参见 §2.4）。

等于比较不会将字符串转换为数字或反之亦然。 因此，"0"==0 求值为 false， t[0] 和 t["0"] 表示表中不同的条目。

运算符 ~= 正好是等于（==）的否定。

顺序运算符的工作方式如下。 如果两个参数都是数字， 则根据它们的数学值进行比较， 与其子类型无关。 否则，如果两个参数都是字符串， 则根据当前区域设置比较它们的值。 否则，Lua 尝试调用 __lt 或 __le 元方法（参见 §2.4）。 比较 a > b 被转换为 b < a， a >= b 被转换为 b <= a。

根据 IEEE 754 标准， 特殊值 NaN 既不小于、也不等于、也不大于任何值，包括其自身。

3.4.5 – 逻辑运算符

Lua 中的逻辑运算符是 and、or 和 not。 与控制结构类似（参见 §3.3.4）， 所有逻辑运算符都将 false 和 nil 视为假， 将其他任何值视为真。

否定运算符 not 始终返回 false 或 true。 合取运算符 and 在其第一个参数的值为 false 或 nil 时返回该参数； 否则，and 返回其第二个参数。 析取运算符 or 在其第一个参数的值不同于 nil 和 false 时返回该参数； 否则，or 返回其第二个参数。 and 和 or 都使用短路求值； 即，只在必要时才对第二个操作数求值。 以下是一些示例： 

     10 or 20            --> 10
     10 or error()       --> 10
     nil or "a"          --> "a"
     nil and 10          --> nil
     false and error()   --> false
     false and nil       --> false
     false or nil        --> nil
     10 and 20           --> 20

3.4.6 – 连接

Lua 中的字符串连接运算符 由两个点（'..'）表示。 如果两个操作数都是字符串或数字， 则数字将被转换为字符串， 格式未指定（参见 §3.4.3）。 否则，调用 __concat 元方法（参见 §2.4）。

3.4.7 – 长度运算符

长度运算符由一元前缀运算符 # 表示。

字符串的长度是其字节数。 （这是每个字符为一个字节时字符串长度的常见含义。）

应用于表的长度运算符 返回该表的一个边界。 表 t 中的边界是满足以下条件的任何非负整数： 

     (border == 0 or t[border] ~= nil) and
     (t[border + 1] == nil or border == math.maxinteger)

换言之， 边界是表中存在的任何正整数索引，其后跟一个不存在的索引， 加上两种边界情况： 零（当索引 1 不存在时）； 以及整数的最大值（当该索引存在时）。 请注意，非正整数的键不会干扰边界。

恰好有一个边界的表称为序列。 例如，表 {10,20,30,40,50} 是一个序列， 因为它只有一个边界（5）。 表 {10,20,30,nil,50} 有两个边界（3 和 5）， 因此它不是序列。 （索引 4 处的 nil 称为空洞。） 表 {nil,20,30,nil,nil,60,nil} 有三个边界（0、3 和 6）， 所以它也不是序列。 表 {} 是边界为 0 的序列。

当 t 是序列时， #t 返回其唯一的边界， 这对应于序列长度的直观概念。 当 t 不是序列时， #t 可以返回其任何边界。 （具体返回哪个取决于表的内部表示细节， 而内部表示又可能取决于表的填充方式 及其非数字键的内存地址。）

表长度的计算 保证最坏情况时间为 O(log n)， 其中 n 是表中最大的整数键。

程序可以通过 __len 元方法修改长度运算符对除字符串外任何值的行为 （参见 §2.4）。

3.4.8 – 优先级

Lua 中的运算符优先级遵循下表， 从低到高： 

     or
     and
     <     >     <=    >=    ~=    ==
     |
     ~
     &
     <<    >>
     ..
     +     -
     *     /     //    %
     unary operators (not   #     -     ~)
     ^

和通常一样， 你可以使用括号来改变表达式的优先级。 连接（'..'）和幂运算（'^'） 运算符是右结合的。 所有其他二元运算符是左结合的。

3.4.9 – 表构造器

表构造器是创建表的表达式。 每次对构造器求值时，都会创建一个新表。 构造器可用于创建空表 或创建表并初始化其某些字段。 构造器的一般语法是 

	tableconstructor ::= ‘{’ [fieldlist] ‘}’
	fieldlist ::= field {fieldsep field} [fieldsep]
	field ::= ‘[’ exp ‘]’ ‘=’ exp | Name ‘=’ exp | exp
	fieldsep ::= ‘,’ | ‘;’

[exp1] = exp2 形式的每个字段向新表添加一个 键为 exp1、值为 exp2 的条目。 name = exp 形式的字段等价于 ["name"] = exp。 exp 形式的字段等价于 [i] = exp，其中 i 是从 1 开始的连续整数； 其他格式的字段不影响此计数。 例如， 

     a = { [f(1)] = g; "x", "y"; x = 1, f(x), [30] = 23; 45 }

等价于 

     do
       local t = {}
       t[f(1)] = g
       t[1] = "x"         -- 1st exp
       t[2] = "y"         -- 2nd exp
       t.x = 1            -- t["x"] = 1
       t[3] = f(x)        -- 3rd exp
       t[30] = 23
       t[4] = 45          -- 4th exp
       a = t
     end

构造器中赋值的顺序是未定义的。 （此顺序仅在存在重复键时才相关。）

如果列表中的最后一个字段具有 exp 形式 且该表达式是多值表达式， 则该表达式返回的所有值将连续进入列表 （参见 §3.4.12）。

字段列表可以有一个可选的尾部分隔符， 作为机器生成代码的便利。

3.4.10 – 函数调用

Lua 中的函数调用具有以下语法： 

	functioncall ::= prefixexp args

在函数调用中， 首先对 prefixexp 和 args 求值。 如果 prefixexp 的值的类型为 function， 则使用给定的参数调用此函数。 否则，如果存在， 则调用 prefixexp 的 __call 元方法： 其第一个参数是 prefixexp 的值， 后跟原始调用参数 （参见 §2.4）。

形式 

	functioncall ::= prefixexp ‘:’ Name args

可用于模拟方法。 调用 v:name(args) 是 v.name(v, args) 的语法糖， 但 v 只被求值一次。

参数具有以下语法： 

	args ::= ‘(’ [explist] ‘)’
	args ::= tableconstructor
	args ::= LiteralString

所有参数表达式在调用之前求值。 形式为 f{fields} 的调用是 f({fields}) 的语法糖； 即参数列表是单个新表。 形式为 f'string' （或 f"string" 或 f[[string]]）的调用 是 f('string') 的语法糖； 即参数列表是单个字面字符串。

形式为 return functioncall 的调用不在 待关闭变量的作用域内，称为尾调用。 Lua 实现正规尾调用 （或正规尾递归）： 在尾调用中， 被调用函数重用调用函数的栈条目。 因此，程序可以执行的嵌套尾调用数量没有限制。 但是，尾调用会擦除关于调用函数的任何调试信息。 请注意，尾调用只在特定语法中发生， 其中 return 以单个函数调用作为参数， 并且它在任何待关闭变量的作用域之外。 此语法使调用函数精确返回 被调用函数的返回值， 没有任何中间操作。 因此，以下示例都不是尾调用： 

     return (f(x))        -- results adjusted to 1
     return 2 * f(x)      -- result multiplied by 2
     return x, f(x)       -- additional results
     f(x); return         -- results discarded
     return x or f(x)     -- results adjusted to 1

3.4.11 – 函数定义

函数定义的语法是 

	functiondef ::= function funcbody
	funcbody ::= ‘(’ [parlist] ‘)’ block end

以下语法糖简化了函数定义： 

	stat ::= function funcname funcbody
	stat ::= local function Name funcbody
	stat ::= global function Name funcbody
	funcname ::= Name {‘.’ Name} [‘:’ Name]

语句 

     function f () body end

翻译为 

     f = function () body end

The statement 

     function t.a.b.c.f () body end

translates to 

     t.a.b.c.f = function () body end

语句 

     local function f () body end

翻译为 

     local f; f = function () body end

而不是 

     local f = function () body end

（这仅在函数体包含对 f 的递归引用时才有区别。） 类似地，语句 

     global function f () body end

翻译为 

     global f; global f = function () body end

第二个 global 使赋值成为初始化， 如果该全局变量已定义，则会引发错误。

冒号语法 用于模拟方法， 向函数添加一个隐式的额外参数 self。 因此，语句 

     function t.a.b.c:f (params) body end

是以下代码的语法糖 

     t.a.b.c.f = function (self, params) body end

函数定义是可执行表达式， 其值的类型为 function。 当 Lua 预编译代码块时， 其所有函数体也被预编译， 但它们尚未创建。 然后，每当 Lua 执行函数定义时， 函数就会被实例化（或闭包）。 此函数实例或闭包 是表达式的最终值。

结果使用 return 语句返回（参见 §3.3.4）。 如果控制流到达函数末尾 而未遇到 return 语句， 则函数无结果返回。

函数可以返回的值数量有一个系统相关的限制。 此限制保证至少为 1000。

参数

参数充当用参数值初始化的局部变量： 

	parlist ::= namelist [‘,’ varargparam] | varargparam
	varargparam ::= ‘...’ [Name]

当调用 Lua 函数时， 它将参数列表调整为 参数列表的长度（参见 §3.4.12）， 除非该函数是可变参数函数， 这由参数列表末尾的三个点（'...'）表示。 可变参数函数不会调整其参数列表； 相反，它收集所有额外参数并通过可变参数表提供给函数。 在该表中， 索引 1、2 等处的值是额外参数， 索引 "n" 处的值是额外参数的数量。

例如，考虑以下定义： 

     function f(a, b) end
     function g(a, b, ...) end
     function r() return 1,2,3 end

那么，我们有以下从参数到参数以及到可变参数表的映射： 

     CALL             PARAMETERS
     
     f(3)             a=3, b=nil
     f(3, 4)          a=3, b=4
     f(3, 4, 5)       a=3, b=4
     f(r(), 10)       a=1, b=10
     f(r())           a=1, b=2
     
     g(3)             a=3, b=nil, va. table ->  {n = 0}
     g(3, 4)          a=3, b=4,   va. table ->  {n = 0}
     g(3, 4, 5, 8)    a=3, b=4,   va. table ->  {5, 8, n = 2}
     g(5, r())        a=5, b=1,   va. table ->  {2, 3, n = 2}

可变参数函数中的可变参数表可以有一个可选名称， 写在三个点之后。 当存在时， 该名称表示引用可变参数表的只读局部变量。 如果可变参数表没有名称， 则只能通过可变参数表达式访问它。

可变参数表达式也写为三个点， 其值是可变参数表中值的列表， 从 1 到索引 "n" 处的整数值。 （因此，如果代码不修改可变参数表， 此列表对应于函数调用中的额外参数。） 此列表的行为类似于具有多个结果的函数的返回值 （参见 §3.4.12）。

作为优化， 如果可变参数表满足某些条件， 代码不会创建实际的表，而是将 索引表达式和可变参数表达式 转换为对内部可变参数数据的访问。 条件如下： 如果可变参数表有名称， 则该名称不是嵌套函数中的上值， 并且仅用作 语法构造 t[exp] 或 t.id 中的基础表。 请注意，匿名可变参数表始终满足这些条件。

3.4.12 – 表达式列表、多值结果和调整

函数调用和可变参数表达式都可以产生多个值。 这些表达式称为多值表达式。

当多值表达式用作表达式列表的最后一个元素时， 该表达式的所有结果都会添加到 表达式列表产生的值列表中。 请注意，在期望表达式列表的位置中的单个表达式 是该（单元素）列表中的最后一个表达式。

以下是 Lua 期望表达式列表的位置：

  • return 语句， 例如 return e1,e2,e3（参见 §3.3.4）。
  • 表构造器， 例如 {e1,e2,e3}（参见 §3.4.9）。
  • 函数调用的参数， 例如 foo(e1,e2,e3)（参见 §3.4.10）。
  • 多重赋值， 例如 a,b,c = e1,e2,e3（参见 §3.3.3）。
  • 局部或全局声明， 类似于多重赋值。
  • 泛型 for 循环中的初始值， 例如 for k in e1,e2,e3 do ... end（参见 §3.3.5。

在最后四种情况下， 表达式列表的值列表 必须调整为特定长度： 调用非可变参数函数时的参数数量 （参见 §3.4.11）、 多重赋值或声明中的变量数量， 以及泛型 for 循环的恰好四个值。 调整遵循以下规则： 如果值多于需要的， 则丢弃多余的值； 如果值少于需要的， 则用 nil 扩展列表。 当表达式列表以多值表达式结尾时， 该表达式的所有结果在调整之前进入值列表。

当多值表达式用作表达式列表中非最后一个元素， 或在语法期望单个表达式的位置时， Lua 将该表达式的结果列表调整为一个元素。 在特定情况下， 语法期望在括号表达式中只有一个表达式； 因此，在多值表达式周围添加括号 强制其恰好产生一个结果。

我们很少需要在语法期望单个表达式的位置 使用可变参数表达式。 （通常在可变参数部分之前添加常规参数 并使用该参数更简单。） 当有这种需求时， 我们建议将可变参数表达式 赋值给单个变量，然后使用该变量 代替它。

以下是多值表达式用法的一些示例。 在所有情况下，当构造需要"第 n 个结果"但不存在这样的结果时， 它使用 nil。 

     print(x, f())      -- prints x and all results from f().
     print(x, (f()))    -- prints x and the first result from f().
     print(f(), x)      -- prints the first result from f() and x.
     print(1 + f())     -- prints 1 added to the first result from f().
     local x = ...      -- x gets the first vararg argument.
     x,y = ...          -- x gets the first vararg argument,
                        -- y gets the second vararg argument.
     x,y,z = w, f()     -- x gets w, y gets the first result from f(),
                        -- z gets the second result from f().
     x,y,z = f()        -- x gets the first result from f(),
                        -- y gets the second result from f(),
                        -- z gets the third result from f().
     x,y,z = f(), g()   -- x gets the first result from f(),
                        -- y gets the first result from g(),
                        -- z gets the second result from g().
     x,y,z = (f())      -- x gets the first result from f(), y and z get nil.
     return f()         -- returns all results from f().
     return x, ...      -- returns x and all received vararg arguments.
     return x,y,f()     -- returns x, y, and all results from f().
     {f()}              -- creates a list with all results from f().
     {...}              -- creates a list with all vararg arguments.
     {f(), 5}           -- creates a list with the first result from f() and 5.

4 – 应用程序接口

本节描述 Lua 的 C API，即宿主程序可用于与 Lua 通信的 C 函数集合。 所有 API 函数以及相关的类型和常量 都在头文件 lua.h 中声明。

即使我们使用“函数”这一术语， API 中的任何功能都可能以宏的形式提供。 除非另有说明， 所有这些宏对每个参数都只求值一次 （第一个参数除外，它始终是 Lua 状态）， 因此不会产生任何隐藏的副作用。

与大多数 C 库一样， Lua API 函数不会检查其参数的有效性或一致性。 但是，你可以通过在编译 Lua 时定义宏 LUA_USE_APICHECK 来改变这种行为。

Lua 库是完全可重入的： 它没有全局变量。 它将所需的所有信息保存在一个动态结构中， 称为 Lua 状态。

每个 Lua 状态有一个或多个线程， 它们对应于独立的、协作的执行线路。 类型 lua_State（尽管其名称如此）指的是一个线程。 （间接地，通过该线程，它也引用与该线程关联的 Lua 状态。）

线程指针必须作为第一个参数传递给 库中的每个函数，但 lua_newstate 除外， 它从零开始创建一个 Lua 状态并返回 新状态中主线程的指针。

4.1 – 栈

Lua 使用虚拟栈来在 C 和 Lua 之间传递值。 栈中的每个元素代表一个 Lua 值 （nil、数字、字符串等）。 API 中的函数可以通过接收到的 Lua 状态参数来访问这个栈。

每当 Lua 调用 C 时，被调用的函数会获得一个新的栈， 它与之前的栈以及仍然活跃的 C 函数的栈是相互独立的。 这个栈最初包含 C 函数的所有参数， 并且 C 函数可以在此存储临时的 Lua 值，还必须将要返回给调用者的结果 压入此栈（参见 lua_CFunction）。

为了方便起见， API 中的大多数查询操作并不遵循严格的栈规则。 相反，它们可以通过使用索引 来引用栈中的任何元素： 正索引表示栈的绝对位置， 从栈底的 1 开始； 负索引表示相对于栈顶的偏移。 更具体地说，如果栈有 n 个元素， 那么索引 1 表示第一个元素 （即最先压入栈的元素） 而索引 n 表示最后一个元素； 索引 -1 也表示最后一个元素 （即栈顶的元素） 而索引 -n 表示第一个元素。

4.1.1 – 栈大小

当使用 Lua API 时， 你需要确保一致性。 特别是， 你需要负责控制栈溢出。 当你调用任何 API 函数时， 你必须确保栈有足够的空间来容纳结果。

上述规则有一个例外： 当你调用一个没有固定返回值数量的 Lua 函数时 （参见 lua_call）， Lua 确保栈有足够的空间容纳所有结果。 但是，它不确保任何额外的空间。 因此，在此类调用之后向栈中压入任何内容之前， 你应该使用 lua_checkstack。

每当 Lua 调用 C 时， 它确保栈至少有 LUA_MINSTACK 个额外元素的空间； 也就是说，你可以安全地向其中压入最多 LUA_MINSTACK 个值。 LUA_MINSTACK 定义为 20， 因此通常你不必担心栈空间， 除非你的代码中有向栈压入元素的循环。 在必要时， 你可以使用 lua_checkstack 函数 来确保栈有足够的空间压入新元素。

4.1.2 – 有效索引和可接受索引

API 中任何接收栈索引的函数 仅适用于有效索引或可接受索引。

有效索引是引用存储可修改 Lua 值的位置的索引。 它包括从 1 到栈顶的栈索引 （1 ≤ abs(index) ≤ top） 加上伪索引， 伪索引表示 C 代码可以访问但不在栈中的某些位置。 伪索引用于访问注册表（参见 §4.3） 和 C 函数的上值（参见 §4.2）。

不需要特定可变位置而只需要值的函数 （例如查询函数） 可以使用可接受索引来调用。 可接受索引可以是任何有效索引， 也可以是栈顶之后、 在栈分配空间内的任何正索引， 即索引不超过栈大小。 （注意 0 永远不是可接受索引。） 大于当前 C 函数中上值 实际数量的上值索引（参见 §4.2） 也是可接受的（但无效）。 除非另有说明， API 中的函数都使用可接受索引。

可接受索引用于避免在查询栈时 对栈顶进行额外的测试。 例如，C 函数可以查询其第三个参数 而无需检查是否存在第三个参数， 即无需检查 3 是否为有效索引。

对于可以使用可接受索引调用的函数， 任何非有效索引都被视为包含 虚拟类型 LUA_TNONE 的值， 其行为类似于 nil 值。

4.1.3 – 字符串指针

API 中的几个函数返回指向栈中 Lua 字符串的指针（const char*）。 （参见 lua_pushfstring、lua_pushlstring、 lua_pushstring 和 lua_tolstring。 另请参阅辅助库中的 luaL_checklstring、luaL_checkstring 和 luaL_tolstring。）

一般来说， Lua 的垃圾回收可以释放或移动内存， 然后使 Lua 状态处理的字符串指针失效。 为了安全地使用这些指针， API 保证栈索引中指向字符串的任何指针 在该索引处的字符串值未从栈中移除时都是有效的。 （但它可以被移动到另一个索引。） 当索引是伪索引（引用上值）时， 指针在对应的调用活跃且 对应的上值未被修改时是有效的。

调试接口中的一些函数 也返回字符串的指针， 即 lua_getlocal、lua_getupvalue、 lua_setlocal 和 lua_setupvalue。 对于这些函数，指针保证在 调用函数活跃且给定的闭包（如果给定的话） 在栈中时是有效的。

除了这些保证之外， 垃圾回收器可以自由地使 任何指向内部字符串的指针失效。

4.2 – C 闭包

当创建一个 C 函数时， 可以将一些值与其关联， 从而创建一个 C 闭包 （参见 lua_pushcclosure）； 这些值称为上值， 在函数每次被调用时都可以访问。

每当调用 C 函数时， 其上值位于特定的伪索引处。 这些伪索引由宏 lua_upvalueindex 生成。 与函数关联的第一个上值位于索引 lua_upvalueindex(1)，以此类推。 访问 lua_upvalueindex(n) 时， 如果 n 大于当前函数的上值数量 （但不大于 256，即闭包中最大上值数量加一）， 会产生一个可接受但无效的索引。

C 闭包也可以更改 其对应上值的值。

4.3 – 注册表

Lua 提供了一个注册表， 一个预定义的表，任何 C 代码都可以使用它来 存储所需的任何 Lua 值。 注册表始终可以通过伪索引 LUA_REGISTRYINDEX 访问。 任何 C 库都可以将数据存储到此表中， 但必须注意选择与其他库 不同的键，以避免冲突。 通常，你应该使用包含库名的字符串作为键， 或者使用指向代码中 C 对象地址的轻量用户数据， 或者由代码创建的任何 Lua 对象。 与变量名一样， 以下划线开头后跟大写字母的字符串键 是 Lua 保留的。

注册表中的整数键被 引用机制使用（参见 luaL_ref）， 并带有一些预定义值。 因此，注册表中的整数键 不得用于其他目的。

当你创建一个新的 Lua 状态时， 其注册表带有一些预定义值。 这些预定义值以在 lua.h 中 定义为常量的整数键进行索引。 定义了以下常量：

  • LUA_RIDX_MAINTHREAD: 在此索引处，注册表包含 状态的主线程。 （主线程是与状态一起创建的那个。）
  • LUA_RIDX_GLOBALS: 在此索引处，注册表包含 全局环境。

4.4 – C 中的错误处理

在内部，Lua 使用 C 的 longjmp 机制来处理错误。 （如果你将 Lua 编译为 C++， Lua 将使用异常； 在源代码中搜索 LUAI_THROW 了解详情。） 当 Lua 遇到任何错误时， 如内存分配错误或类型错误， 它会引发一个错误； 即执行长跳转。 受保护环境使用 setjmp 设置恢复点； 任何错误都会跳转到最近的活动恢复点。

在 C 函数内部，你可以通过调用 lua_error 来显式地引发错误。

API 中的大多数函数都可以引发错误， 例如由于内存分配错误。 每个函数的文档都指出了 它是否可以引发错误。

如果错误发生在任何受保护环境之外， Lua 会调用一个panic 函数（参见 lua_atpanic） 然后调用 abort， 从而退出宿主应用程序。 你的 panic 函数可以通过永不返回 来避免此退出 （例如，长跳转到 Lua 之外的你自己的恢复点）。

panic 函数， 顾名思义， 是最后的手段。 程序应该避免使用它。 一般来说， 当 Lua 使用 Lua 状态调用 C 函数时， 它可以在该 Lua 状态上做任何它想做的事， 因为它应该已经被保护了。 但是， 当 C 代码操作其他 Lua 状态时 （例如，函数的 Lua 状态参数、 存储在注册表中的 Lua 状态，或 lua_newthread 的结果）， 它应该只在不会引发错误的 API 调用中使用它们。

panic 函数的运行方式类似于消息处理器（参见 §2.3）； 特别是，错误对象在栈顶。 但是，不保证栈空间。 要向栈中压入任何内容， panic 函数必须首先检查可用空间（参见 §4.1.1）。

4.4.1 – 状态码

API 中报告错误的几个函数使用以下 状态码来指示不同类型的错误或其他条件：

  • LUA_OK (0): 无错误。
  • LUA_ERRRUN: 运行时错误。
  • LUA_ERRMEM: 内存分配错误。 对于此类错误，Lua 不会调用消息处理器。
  • LUA_ERRERR: 运行消息处理器时因另一个栈溢出而导致栈溢出。 通常，此错误是在运行消息处理器时 由于某些其他错误而产生的结果。 消息处理器中的错误将再次调用处理器， 这将再次生成错误，依此类推， 直到此循环耗尽栈并导致此错误。
  • LUA_ERRSYNTAX: 预编译期间的语法错误 或二进制块中的格式错误。
  • LUA_YIELD: 线程（协程）让出。
  • LUA_ERRFILE: 与文件相关的错误； 例如，无法打开或读取文件。

这些常量在头文件 lua.h 中定义。

4.5 – C 中的让出处理

在内部，Lua 使用 C 的 longjmp 机制来让出协程。 因此，如果 C 函数 foo 调用一个 API 函数 而该 API 函数让出 （直接或间接地通过调用另一个让出的函数）， Lua 无法再返回到 foo， 因为 longjmp 会将其帧从 C 栈中移除。

为了避免这类问题， Lua 在尝试跨 API 调用让出时会引发错误， 除了三个函数： lua_yieldk、lua_callk 和 lua_pcallk。 所有这些函数都接收一个续延函数 （作为名为 k 的参数）以便在让出后继续执行。

我们需要设定一些术语来解释续延。 我们有一个从 Lua 调用的 C 函数， 我们称之为原始函数。 这个原始函数然后调用 C API 中的这三个函数之一， 我们称之为被调函数， 然后让出当前线程。 当被调函数是 lua_yieldk 时， 或者当被调函数是 lua_callk 或 lua_pcallk 且它们调用的函数让出时，就会发生这种情况。

假设运行中的线程在执行被调函数时让出。 线程恢复后， 它最终会完成被调函数的执行。 但是， 被调函数无法返回到原始函数， 因为其在 C 栈中的帧已被让出销毁。 相反，Lua 调用一个续延函数， 它作为参数传递给被调函数。 顾名思义， 续延函数应该继续原始函数的任务。

作为说明，考虑以下函数： 

     int original_function (lua_State *L) {
       ...     /* code 1 */
       status = lua_pcall(L, n, m, h);  /* calls Lua */
       ...     /* code 2 */
     }

现在我们想要允许 由 lua_pcall 运行的 Lua 代码让出。 首先，我们可以像这样重写我们的函数： 

     int k (lua_State *L, int status, lua_KContext ctx) {
       ...  /* code 2 */
     }
     
     int original_function (lua_State *L) {
       ...     /* code 1 */
       return k(L, lua_pcall(L, n, m, h), ctx);
     }

在上面的代码中， 新函数 k 是一个 续延函数（类型为 lua_KFunction）， 它应该执行原始函数 在调用 lua_pcall 之后所做的所有工作。 现在，我们必须通知 Lua，如果由 lua_pcall 执行的 Lua 代码以某种方式被中断 （错误或让出）， 它必须调用 k， 因此我们像这样重写代码， 将 lua_pcall 替换为 lua_pcallk： 

     int original_function (lua_State *L) {
       ...     /* code 1 */
       return k(L, lua_pcallk(L, n, m, h, ctx2, k), ctx1);
     }

注意对续延的外部显式调用： Lua 只在需要时才会调用续延，即 在错误或从让出恢复的情况下。 如果被调函数正常返回而从未让出， lua_pcallk（和 lua_callk）也会正常返回。 （当然，在这种情况下，你可以在原始函数内部直接执行等效的工作。）

除了 Lua 状态之外， 续延函数还有两个参数： 调用的最终状态和最初传递给 lua_pcallk 的上下文值（ctx）。 Lua 不使用此上下文值； 它只是将此值从原始函数传递到续延函数。 对于 lua_pcallk， 状态与 lua_pcallk 返回的值相同， 但在让出后执行时为 LUA_YIELD （而不是 LUA_OK）。 对于 lua_yieldk 和 lua_callk， 当 Lua 调用续延时状态始终为 LUA_YIELD。 （对于这两个函数， Lua 在错误时不会调用续延， 因为它们不处理错误。） 类似地，使用 lua_callk 时， 你应该以 LUA_OK 作为状态 调用续延函数。 （对于 lua_yieldk，直接调用续延函数没有太大意义， 因为 lua_yieldk 通常不会返回。）

Lua 将续延函数视为原始函数。 续延函数接收来自原始函数的相同 Lua 栈， 处于与被调函数返回时相同的状态。 （例如，在 lua_callk 之后， 函数及其参数从栈中移除并替换为调用结果。） 它还具有相同的上值。 它返回的任何内容都由 Lua 作为原始函数的返回值处理。

4.6 – 函数和类型

我们按字母顺序列出 C API 中的所有函数和类型。 每个函数都有如下指示器： [-o, +p, x]

第一个字段 o 是函数从栈中弹出的元素数量。 第二个字段 p 是函数压入栈的元素数量。 （任何函数在弹出参数后总是压入其结果。） x|y 形式的字段意味着函数可以压入（或弹出） x 或 y 个元素， 取决于情况； 问号 '?' 意味着 我们仅通过查看参数无法知道函数弹出/压入多少元素。 （例如，它们可能取决于栈中的内容。） 第三个字段 x 告诉函数是否可能引发错误： '-' 表示函数从不引发任何错误； 'm' 表示函数可能仅引发内存不足错误； 'v' 表示函数可能引发文中解释的错误； 'e' 表示函数可以运行任意 Lua 代码， 直接或通过元方法， 因此可能引发任何错误。

lua_Alloc

typedef void * (*lua_Alloc) (void *ud,
                             void *ptr,
                             size_t osize,
                             size_t nsize);

Lua 状态使用的内存分配器函数的类型。 分配器函数必须提供类似于 realloc 的功能， 但不完全相同。 其参数为 ud，传递给 lua_newstate 的不透明指针； ptr，指向正在分配/重新分配/释放的块的指针； osize，块的原始大小或关于正在分配的内容的某些代码； 以及 nsize，块的新大小。

当 ptr 不为 NULL 时， osize 是 ptr 指向的块的大小， 即分配或重新分配时给出的大小。

当 ptr 为 NULL 时， osize 编码 Lua 正在分配的对象类型。 当（且仅当）Lua 正在创建该类型的新对象时， osize 是 LUA_TSTRING、LUA_TTABLE、LUA_TFUNCTION、 LUA_TUSERDATA 或 LUA_TTHREAD 之一。 当 osize 是其他值时， Lua 正在为其他东西分配内存。

Lua 对分配器函数假设以下行为：

当 nsize 为零时， 分配器必须表现得像 free， 然后返回 NULL。

当 nsize 不为零时， 分配器必须表现得像 realloc。 特别是，分配器当且仅当无法满足请求时 才返回 NULL。

以下是分配器函数的一个简单实现， 对应于辅助库中的 luaL_alloc 函数。 

     void *luaL_alloc (void *ud, void *ptr, size_t osize,
                                            size_t nsize) {
       (void)ud;  (void)osize;  /* not used */
       if (nsize == 0) {
         free(ptr);
         return NULL;
       }
       else
         return realloc(ptr, nsize);
     }

注意 ISO C 确保 free(NULL) 不产生任何效果， 且 realloc(NULL,size) 等同于 malloc(size)。

lua_call

[-(nargs+1), +nresults, e] 

void lua_call (lua_State *L, int nargs, int nresults);

调用一个函数。 与常规 Lua 调用一样， lua_call 遵循 __call 元方法。 因此，这里的\u201c函数\u201d 指的是任何可调用值。

要执行调用，你必须使用以下协议： 首先，将要调用的函数压入栈中； 然后，按正序压入调用的参数； 即第一个参数最先压入。 最后调用 lua_call； nargs 是你压入栈的参数数量。 当函数返回时， 所有参数和函数值都被弹出， 调用结果被压入栈中。 结果数量被调整为 nresults， 除非 nresults 是 LUA_MULTRET， 这会使函数的所有结果都被压入。 在第一种情况下（显式结果数量）， 调用者必须确保栈有空间 容纳返回值。 在第二种情况下（所有结果）， Lua 确保返回值适合栈空间， 但它不确保栈中有任何额外空间。 函数结果按正序压入栈中 （第一个结果最先压入）， 因此调用后最后一个结果在栈顶。

nresults 的最大值为 250。

调用和运行函数时的任何错误都会向上传播 （通过 longjmp）。

以下示例展示了宿主程序如何执行 与此 Lua 代码等效的操作： 

     a = f("how", t.x, 14)

C 代码如下： 

     lua_getglobal(L, "f");                  /* function to be called */
     lua_pushliteral(L, "how");                       /* 1st argument */
     lua_getglobal(L, "t");                    /* table to be indexed */
     lua_getfield(L, -1, "x");        /* push result of t.x (2nd arg) */
     lua_remove(L, -2);                  /* remove 't' from the stack */
     lua_pushinteger(L, 14);                          /* 3rd argument */
     lua_call(L, 3, 1);     /* call 'f' with 3 arguments and 1 result */
     lua_setglobal(L, "a");                         /* set global 'a' */

注意上面的代码是平衡的： 在代码末尾，栈恢复到原始配置。 这被认为是良好的编程实践。

lua_CFunction

typedef int (*lua_CFunction) (lua_State *L);

C 函数的类型。

为了与 Lua 正确通信， C 函数必须使用以下协议， 它定义了参数和结果的传递方式： C 函数从栈中按正序接收来自 Lua 的参数 （第一个参数最先压入）。 因此，当函数开始时， lua_gettop(L) 返回函数接收的参数数量。 第一个参数（如果有的话）在索引 1 处， 最后一个参数在索引 lua_gettop(L) 处。 要将值返回给 Lua，C 函数只需将它们压入栈中， 按正序（第一个结果最先压入）， 并在 C 中返回结果数量。 结果下方栈中的任何其他值 都会被 Lua 正确丢弃。 与 Lua 函数一样，Lua 调用的 C 函数 也可以返回多个结果。

作为示例，以下函数接收可变数量的 数值参数并返回它们的平均值和总和： 

     static int foo (lua_State *L) {
       int n = lua_gettop(L);    /* number of arguments */
       lua_Number sum = 0.0;
       int i;
       for (i = 1; i <= n; i++) {
         if (!lua_isnumber(L, i)) {
           lua_pushliteral(L, "incorrect argument");
           lua_error(L);
         }
         sum += lua_tonumber(L, i);
       }
       lua_pushnumber(L, sum/n);        /* first result */
       lua_pushnumber(L, sum);         /* second result */
       return 2;                   /* number of results */
     }

       printf("%s - %s\n",
              lua_typename(L, lua_type(L, -2)),

       lua_pop(L, 1);

lua_pcall

[-(nargs + 1), +(nresults|1), –] 

int lua_pcall (lua_State *L, int nargs, int nresults, int msgh);

在受保护模式下调用函数（或可调用对象）。

nargs 和 nresults 的含义与 lua_call 中相同。 如果调用期间没有错误， lua_pcall 的行为与 lua_call 完全相同。 但是，如果有任何错误， lua_pcall 会捕获它， 在栈上推送一个值（错误对象）， 并返回错误码。 与 lua_call 一样， lua_pcall 总是从栈中移除函数 及其参数。

lua_yieldk

[-?, +?, v]

                int nresults,
                lua_KContext ctx,
                lua_KFunction k);

让出协程（线程）。

When a C function calls lua_yieldk, the running coroutine suspends its execution, and the call to lua_resume that started this coroutine returns. The parameter nresults is the number of values from the stack that will be passed as results to lua_resume.

When the coroutine is resumed again, Lua calls the given continuation function k to continue the execution of the C function that yielded (see §4.5). This continuation function receives the same stack from the previous function, with the n results removed and replaced by the arguments passed to lua_resume. Moreover, the continuation function receives the value ctx that was passed to lua_yieldk.

通常，此函数不会返回； 当协程最终恢复时， 它继续执行续延函数。 但是，有一个特殊情况， 即当此函数从行或计数钩子内部调用时 （参见 §4.7）。 In that case, lua_yieldk should be called with no continuation (probably in the form of lua_yield) and no results, and the hook should return immediately after the call. Lua 将让出， 当协程再次恢复时， 它将继续正常执行触发钩子的（Lua）函数。

如果从此线程调用此函数 且存在没有续延函数的挂起 C 调用 （称为C 调用边界）， 或者从不在 resume 内部运行的线程调用 （通常是主线程）， 此函数可能引发错误。

4.7 – The Debug Interface

Lua 没有内置的调试设施。 相反，它通过函数和钩子 提供了一个特殊的接口。 此接口允许构建不同类型的调试器、 性能分析器和其他需要从解释器获取 “内部信息”的工具。

lua_Debug

typedef struct lua_Debug {
  int event;
  const char *name;           /* (n) */
  const char *namewhat;       /* (n) */
  const char *what;           /* (S) */
  const char *source;         /* (S) */
  size_t srclen;              /* (S) */
  int currentline;            /* (l) */
  int linedefined;            /* (S) */
  int lastlinedefined;        /* (S) */
  unsigned char nups;         /* (u) number of upvalues */
  unsigned char nparams;      /* (u) number of parameters */
  char isvararg;              /* (u) */
  unsigned char extraargs;    /* (t) number of extra arguments */
  char istailcall;            /* (t) */
  int ftransfer;              /* (r) index of first value transferred */
  int ntransfer;              /* (r) number of transferred values */
  char short_src[LUA_IDSIZE]; /* (S) */
  /* private part */
  other fields
} lua_Debug;

用于携带关于函数或活动记录的 不同信息的结构。 lua_getstack 仅填充此结构的 私有部分，供以后使用。 要用有用的信息填充 lua_Debug 的其他字段， 你必须使用适当的参数调用 lua_getinfo。 (Specifically, to get a field, you must add the letter between parentheses in the field's comment to the parameter what of lua_getinfo.)

lua_Debug 的字段具有以下含义：

  • source: the source of the chunk that created the function. If source starts with a '@', it means that the function was defined in a file where the file name follows the '@'. If source starts with a '=', the remainder of its contents describes the source in a user-dependent manner. Otherwise, the function was defined in a string where source is that string.
  • srclen: The length of the string source.
  • short_src: source 的“可打印”版本，用于错误消息。
  • linedefined: 函数定义开始的行号。
  • lastlinedefined: 函数定义结束的行号。
  • what: 如果函数是 Lua 函数则字符串为 "Lua"， 如果是 C 函数则为 "C"， 如果是块的主部分则为 "main"。
  • currentline: 给定函数正在执行的当前行。 当没有行信息可用时， currentline 设置为 -1。
  • name: 给定函数的合理名称。 因为 Lua 中的函数是一等值， 它们没有固定的名称： 有些函数可以是多个全局变量的值， 而其他函数只能存储在表字段中。 lua_getinfo 函数检查函数的调用方式 以找到合适的名称。 如果找不到名称， 则 name 设置为 NULL。
  • namewhat: 解释 name 字段。 namewhat 的值可以是 "global"、"local"、"upvalue"、 "field"、""（空字符串），以及一些其他选项， 取决于函数的调用方式。 （当没有其他选项适用时，Lua 使用空字符串。）
  • istailcall: 如果此函数调用是通过尾调用调用的则为 true。 在这种情况下，此级别的调用者不在栈中。
  • extraargs: The number of extra arguments added by the call to functions called through __call metamethods. (Each __call metavalue adds a single extra argument, the object being called, but there may be a chain of __call metavalues.)
  • nups: 函数的上值数量。
  • nparams: 函数的参数数量 （C 函数始终为 0）。
  • isvararg: 如果函数是可变参数函数则为 true （C 函数始终为 true）。
  • ftransfer: the index in the stack of the first value being "transferred", that is, parameters in a call or return values in a return. (The other values are in consecutive indices.) Using this index, you can access and modify these values through lua_getlocal and lua_setlocal. This field is only meaningful during a call hook, denoting the first parameter, or a return hook, denoting the first value being returned. (For call hooks, this value is always 1.)
  • ntransfer: 正在传输的值的数量（参见上一项）。 （对于 Lua 函数的调用， 此值始终等于 nparams。）

To get information about a function invocation, the parameter ar must be a valid activation record that was filled by a previous call to lua_getstack or given as argument to a hook (see lua_Hook).

To get information about a function, you push it onto the stack and start the what string with the character '>'. (In that case, lua_getinfo pops the function from the top of the stack.) For instance, to know in which line a function f was defined, you can write the following code: 

     lua_Debug ar;
     lua_getglobal(L, "f");  /* get global 'f' */
     lua_getinfo(L, ">S", &ar);
     printf("%d\n", ar.linedefined);

Each character in the string what selects some fields of the structure ar to be filled or a value to be pushed on the stack. (These characters are also documented in the declaration of the structure lua_Debug, between parentheses in the comments following each field.)

5 – 辅助库

辅助库提供了若干便捷函数， 用于在 C 与 Lua 之间进行交互。 基本 API 提供了 C 与 Lua 之间所有交互的底层函数， 而辅助库则为一些常见任务提供了更高层次的函数。

辅助库中的所有函数和类型 都在头文件 lauxlib.h 中定义， 并带有 luaL_ 前缀。

辅助库中的所有函数都构建在 基本 API 之上， 因此它们所提供的功能都可以通过基本 API 来完成。 尽管如此，使用辅助库可以 使你的代码更加一致。

辅助库中的若干函数在内部使用了 一些额外的栈空间。 当辅助库中的函数使用的栈空间少于五个槽位时， 它不会检查栈的大小； 它直接假设有足够的栈空间。

辅助库中的若干函数用于 检查 C 函数的参数。 由于错误消息是针对参数格式化的 （例如"bad argument #1"）， 你不应该将这些函数用于其他栈值。

名为 luaL_check* 的函数 在检查不满足时总是会引发错误。

typedef struct luaL_Buffer luaL_Buffer;

字符串缓冲区允许 C 代码逐步构建 Lua 字符串。 其使用模式如下：

  • 首先声明一个 luaL_Buffer 类型的变量 b。
  • 然后使用调用 luaL_buffinit(L,&b) 对其进行初始化。
  • 接着通过调用任何 luaL_add* 函数 向缓冲区添加字符串片段。
  • 最后调用 luaL_pushresult(&b) 完成操作。 此调用会将最终的字符串留在栈顶。

如果你事先知道结果字符串的最大长度， 可以按如下方式使用缓冲区：

  • 首先声明一个 luaL_Buffer 类型的变量 b。
  • 然后使用调用 luaL_buffinitsize(L,&b,sz) 对其进行初始化并预分配大小为 sz 的空间。
  • 接着在该空间中生成字符串。
  • 最后调用 luaL_pushresultsize(&b,sz)， 其中 sz 是复制到该空间的结果字符串的总长度 （可能小于或等于预分配的大小）。

在正常操作期间， 字符串缓冲区会使用可变数量的栈槽位。 因此，在使用缓冲区时，你不能假定自己知道栈顶的位置。 你可以在缓冲区操作的连续调用之间使用栈， 只要这种使用是平衡的； 也就是说， 当你调用缓冲区操作时， 栈的级别应该与上次缓冲区操作之后的级别相同。 （此规则的唯一例外是 luaL_addvalue。） 在调用 luaL_pushresult 之后， 栈会恢复到缓冲区初始化时的级别， 并在栈顶加上最终的字符串。

6 – 标准库

标准 Lua 库提供了许多实用函数， 这些函数通过 C API 用 C 语言实现。 其中一些函数为语言提供基础服务 （例如 type 和 getmetatable）； 另一些提供对外部服务的访问（例如 I/O）； 还有一些虽然可以在 Lua 中实现， 但出于不同原因而选择用 C 来实现 （例如 table.sort）。

所有库都通过官方 C API 实现， 并作为独立的 C 模块提供。 除非另有说明， 这些库函数不会调整其参数数量以匹配预期的参数。 例如，文档中记为 foo(arg) 的函数 不应在没有参数的情况下调用。

标记 fail 表示一个假值，代表某种失败。 （目前 fail 等于 nil， 但在未来版本中可能会改变。 建议始终使用 (not status) 而不是 (status == nil) 来测试这些函数的成功与否。）

目前，Lua 拥有以下标准库：

  • 基础库（§6.2）；
  • 协程库（§6.3）；
  • 包库（§6.4）；
  • 字符串操作（§6.5）；
  • 基本 UTF-8 支持（§6.6）；
  • 表操作（§6.7）；
  • 数学函数（§6.8）（sin、log 等）；
  • 输入和输出（§6.9）；
  • 操作系统设施（§6.10）；
  • 调试设施（§6.11）。

除基础库和包库外， 每个库的所有函数都作为全局表的字段 或其对象的方法提供。

6.1 – 在 C 代码中加载库

C 宿主程序必须显式地将标准库加载到状态中， 如果它希望脚本能够使用这些库的话。 为此，宿主程序可以调用 luaL_openlibs 函数。 或者，宿主可以使用 luaL_openselectedlibs 来选择要打开的库。 这两个函数都在头文件 lualib.h 中声明。

独立解释器 lua（参见 §7） 已经打开了所有标准库。

luaL_openlibs

[-0, +0, e] 

void luaL_openlibs (lua_State *L);

将所有标准 Lua 库打开到给定的状态中。

luaL_openselectedlibs

[-0, +0, e] 

void luaL_openselectedlibs (lua_State *L, int load, int preload);

打开（加载）并预加载选定的标准库到状态 L 中。 （预加载是指将库的加载器添加到 package.preload 表中， 以便程序后续可以通过 require 来加载该库。 请注意 require 本身 是由 package 库提供的。 如果程序没有加载该库， 它将无法 require 任何东西。）

整数 load 选择要加载哪些库； 整数 preload 选择在未加载的库中要预加载哪些。 两者都是通过按位或操作由以下常量形成的掩码：

  • LUA_GLIBK ：基础库。
  • LUA_LOADLIBK ：包库。
  • LUA_COLIBK ：协程库。
  • LUA_STRLIBK ：字符串库。
  • LUA_UTF8LIBK ：UTF-8 库。
  • LUA_TABLIBK ：表库。
  • LUA_MATHLIBK ：数学库。
  • LUA_IOLIBK ：I/O 库。
  • LUA_OSLIBK ：操作系统库。
  • LUA_DBLIBK ：调试库。

6.2 – 基础函数

基础库为 Lua 提供核心函数。 如果你的应用程序中没有包含这个库， 你应该仔细检查是否需要为其中某些功能提供实现。

assert (v [, message])

如果其参数 v 的值为假（即 nil 或 false）， 则引发错误； 否则返回其所有参数。 在出错的情况下， message 是错误对象； 当省略时，默认为 "assertion failed!"

collectgarbage ([opt [, arg]])

此函数是垃圾收集器的通用接口。 它根据第一个参数 opt 执行不同的功能：

  • "collect"： 执行一个完整的垃圾收集周期。 这是默认选项。
  • "stop"： 停止垃圾收集器的自动执行。 收集器将仅在显式调用时运行， 直到调用 restart 来重启它。
  • "restart"： 重启垃圾收集器的自动执行。
  • "count"： 返回 Lua 使用的总内存（以 Kbytes 为单位）。 该值包含小数部分， 因此乘以 1024 后可得到 Lua 使用的精确字节数。
  • "step"： 执行一个垃圾收集步骤。 此选项后面可以跟一个额外的参数， 即一个整数，表示步长大小。

    如果大小是正数 n， 收集器的行为就好像分配了 n 个新字节。 如果大小为零， 收集器执行一个基本步骤。 在增量模式下， 基本步骤对应于当前步长。 在分代模式下， 基本步骤执行一次完整的小型收集， 或者如果收集器已调度了一个增量步骤， 则执行该增量步骤。

    在增量模式下， 如果步骤完成了一个收集周期，函数返回 true。 在分代模式下， 如果步骤完成了一次大型收集，函数返回 true。

  • "isrunning"： 返回一个布尔值，指示收集器是否正在运行 （即未被停止）。
  • "incremental"： 将收集器模式更改为增量模式并返回之前的模式。
  • "generational"： 将收集器模式更改为分代模式并返回之前的模式。
  • "param"： 更改和/或检索收集器参数的值。 此选项必须后跟一个或两个额外参数： 被更改或检索的参数名称（字符串） 以及该参数的可选新值， 一个范围在 [0,100000] 内的整数。 第一个参数必须具有以下值之一：
    • "minormul"：小型乘数。
    • "majorminor"：大-小乘数。
    • "minormajor"：小-大乘数。
    • "pause"：垃圾收集器暂停值。
    • "stepmul"：步长乘数。
    • "stepsize"：步长大小。

    该调用始终返回参数的先前值。 如果调用未提供新值， 则该值保持不变。

    Lua 以压缩格式存储这些值， 因此作为先前值返回的值可能不是 最后设置的值的精确值。

有关垃圾收集及其中一些选项的更多详情， 请参见 §2.5。

此函数不应由终结器调用。

dofile ([filename])

打开指定文件并将其内容作为 Lua 代码块执行， 返回该代码块返回的所有值。 不带参数调用时， dofile 执行标准输入（stdin）的内容。 在出错情况下，dofile 将错误传播给调用者。 （即 dofile 不在保护模式下运行。）

error (message [, level])

以 message 作为错误对象引发错误（参见 §2.3）。 此函数永远不返回。

通常，如果消息是字符串，error 会在消息开头添加一些关于错误位置的信息。 level 参数指定如何获取错误位置。 当 level 为 1（默认值）时，错误位置是调用 error 函数的地方。 Level 2 指向调用 error 的函数被调用的地方；以此类推。 传递 level 0 可避免在消息中添加错误位置信息。

_G

一个全局变量（不是函数）， 保存全局环境（参见 §2.2）。 Lua 本身不使用此变量； 更改其值不会影响任何环境，反之亦然。

getmetatable (object)

如果 object 没有元表，则返回 nil。 否则， 如果对象的元表有一个 __metatable 字段， 则返回关联的值。 否则返回给定对象的元表。

ipairs (t)

返回三个值（一个迭代器函数、值 t 和 0）， 使得构造 

     for i,v in ipairs(t) do body end

将遍历键值对 (1,t[1])、(2,t[2])、...， 直到第一个缺失的索引。

load (chunk [, chunkname [, mode [, env]]])

加载一个代码块。

如果 chunk 是字符串，则代码块就是该字符串。 如果 chunk 是函数， load 会反复调用它来获取代码块的片段。 对 chunk 的每次调用必须返回一个字符串， 该字符串与之前的结果连接。 返回空字符串、nil 或无值表示代码块结束。

如果没有语法错误， load 将编译后的代码块作为函数返回； 否则，它返回 fail 加上错误消息。

当你加载一个主代码块时， 结果函数始终恰好有一个上值， 即 _ENV 变量（参见 §2.2）。 但是， 当你加载由函数创建的二进制代码块（参见 string.dump）时， 结果函数可以有任意数量的上值， 并且不保证其第一个上值是 _ENV 变量。 （非主函数甚至可能没有 _ENV 上值。）

无论如何，如果结果函数有任何上值， 其第一个上值被设置为 env 的值（如果给出了该参数）， 否则设置为全局环境的值。 其他上值初始化为 nil。 所有上值都是全新的，即它们不与任何其他函数共享。

chunkname 用作错误消息和调试信息（参见 §4.7）中代码块的名称。 省略时， 如果 chunk 是字符串则默认为 chunk， 否则默认为 "=(load)"。

字符串 mode 控制代码块是否可以是文本或二进制 （即预编译的代码块）。 它可以是字符串 "b"（仅二进制代码块）、 "t"（仅文本代码块）， 或 "bt"（二进制和文本均可）。 默认值为 "bt"。

Lua 不检查二进制代码块的一致性。 恶意构造的二进制代码块可能导致解释器崩溃。 你可以使用 mode 参数来防止加载二进制代码块。

loadfile ([filename [, mode [, env]]])

类似于 load， 但从文件 filename 获取代码块， 如果未给出文件名，则从标准输入获取。

next (table [, index])

允许程序遍历表的所有字段。 其第一个参数是一个表，第二个参数 是该表中的一个索引。 调用 next 返回表的下一个索引 及其关联的值。 当以 nil 作为第二个参数调用时， next 返回一个初始索引 及其关联的值。 当以最后一个索引调用时， 或在空表中以 nil 调用时， next 返回 nil。 如果第二个参数不存在，则解释为 nil。 特别是， 你可以使用 next(t) 来检查表是否为空。

枚举索引的顺序是未指定的， 即使是数值索引也是如此。 （要按数值顺序遍历表， 请使用数值 for 循环。）

在遍历表期间，你不应该为不存在的字段赋值。 但是你可以修改现有的字段。 特别是，你可以将现有字段设置为 nil。

pairs (t)

如果 t 有一个元方法 __pairs， 则以 t 作为参数调用它，并返回调用的前四个结果。

否则， 返回 next 函数、表 t 和两个 nil 值， 使得构造 

     for k,v in pairs(t) do body end

将遍历表 t 的所有键值对。

有关在遍历表期间修改表的注意事项， 请参见函数 next。

pcall (f [, arg1, ···])

以保护模式调用函数 f， 传入给定的参数。 这意味着 f 内部的任何错误都不会被传播； 相反，pcall 捕获错误并返回状态码。 其第一个结果是状态码（布尔值）， 如果调用成功没有错误则为 true。 在这种情况下，pcall 还会返回调用的所有结果， 紧随第一个结果之后。 在任何错误情况下，pcall 返回 false 加上错误对象。 注意 pcall 捕获的错误不会调用消息处理器。

print (···)

接收任意数量的参数， 并将其值打印到 stdout， 按照与 tostring 相同的规则 将每个参数转换为字符串。

函数 print 不用于格式化输出， 而只是显示值的快捷方式， 例如用于调试。 要完全控制输出， 请使用 string.format 和 io.write。

rawequal (v1, v2)

检查 v1 是否等于 v2， 不调用 __eq 元方法。 返回一个布尔值。

rawget (table, index)

获取 table[index] 的真实值， 不使用 __index 元值。 table 必须是一个表； index 可以是任何值。

rawlen (v)

返回对象 v 的长度， v 必须是表或字符串， 不调用 __len 元方法。 返回一个整数。

rawset (table, index, value)

将 table[index] 的真实值设置为 value， 不使用 __newindex 元值。 table 必须是一个表， index 是除 nil 和 NaN 以外的任何值， value 是任何 Lua 值。

此函数返回 table。

select (index, ···)

如果 index 是数字， 返回 index 号参数之后的所有参数； 负数从末尾索引（-1 是最后一个参数）。 否则，index 必须是字符串 "#"， select 返回它接收到的额外参数的总数。

setmetatable (table, metatable)

设置给定表的元表。 如果 metatable 是 nil， 则移除给定表的元表。 如果原始元表有一个 __metatable 字段， 则引发错误。

此函数返回 table。

要从 Lua 代码更改其他类型的元表， 你必须使用调试库（§6.11）。

tonumber (e [, base])

不带 base 调用时， tonumber 尝试将其参数转换为数字。 如果参数已经是数字或 可转换为数字的字符串， 则 tonumber 返回该数字； 否则返回 fail。

字符串的转换可以产生整数或浮点数， 遵循 Lua 的词法约定（参见 §3.1）。 字符串可以有前导和尾随空格以及符号。

带 base 调用时， e 必须是一个字符串， 被解释为该进制下的整数字面量。 基数可以是 2 到 36 之间的任何整数（含）。 在 10 以上的基数中，字母 'A'（大写或小写） 代表 10，'B' 代表 11，以此类推， 'Z' 代表 35。 如果字符串 e 在给定基数下不是有效的数字， 则函数返回 fail。

tostring (v)

接收任意类型的值并将其转换为人类可读格式的字符串。

如果 v 的元表有一个 __tostring 字段， 则 tostring 调用相应的值， 以 v 作为参数， 并使用调用的结果作为其结果。 否则，如果 v 的元表有一个 __name 字段 且该字段是字符串值， tostring 可能在最终结果中使用该字符串。

要完全控制数字的转换方式， 请使用 string.format。

type (v)

返回其唯一参数的类型，编码为字符串。 此函数的可能结果为 "nil"（字符串，不是值 nil）， "number"， "string"， "boolean"， "table"， "function"， "thread" 和 "userdata"。

_VERSION

一个全局变量（不是函数）， 保存包含运行中 Lua 版本的字符串。 本变量的当前值是 "Lua 5.5"。

warn (msg1, ···)

发出一条警告，消息由所有参数（应该是字符串）的连接组成。

按照约定， 以 '@' 开头的单段消息是一个控制消息， 它是发送给警告系统本身的消息。 特别是，Lua 中的标准警告函数 识别控制消息 "@off"（停止发出警告） 和 "@on"（（重新）开始发出警告）； 它忽略未知的控制消息。

xpcall (f, msgh [, arg1, ···])

此函数类似于 pcall， 不同之处在于它设置了一个新的消息处理器 msgh。

6.3 – 协程操作

此库包含操作协程的操作， 它们位于表 coroutine 中。 有关协程的一般描述，请参见 §2.6。

coroutine.close ([co])

关闭协程 co， 即关闭所有待关闭的变量， 并将协程置于死亡状态。 co 的默认值是正在运行的协程。

给定的协程必须是死亡的、挂起的， 或者正在运行的协程。 对于正在运行的协程， 此函数不会返回。 相反，（重新）启动协程的 resume 会返回。

对于其他协程， 在出错情况下 （无论是停止协程的原始错误还是关闭方法中的错误）， 此函数返回 false 加上错误对象； 否则返回 true。

coroutine.create (f)

创建一个新协程，主体为 f。 f 必须是一个函数。 返回这个新协程， 一个类型为 "thread" 的对象。

coroutine.isyieldable ([co])

当协程 co 可以让出时返回 true。 co 的默认值是正在运行的协程。

如果协程不是主线程并且 不在不可让出的 C 函数内部， 则该协程是可让出的。

coroutine.resume (co [, val1, ···])

启动或继续协程 co 的执行。 第一次恢复协程时， 它开始运行其主体。 值 val1、... 作为参数传递给主体函数。 如果协程已经让出， resume 重新启动它； 值 val1、... 作为 yield 的结果传递。

如果协程在运行过程中没有错误， resume 返回 true 加上传递给 yield 的所有值 （当协程让出时）或主体函数返回的所有值 （当协程终止时）。 如果有任何错误， resume 返回 false 加上错误消息。

coroutine.running ()

返回正在运行的协程加上一个布尔值， 当正在运行的协程是主协程时为 true。

coroutine.status (co)

以字符串形式返回协程 co 的状态： "running"，如果协程正在运行 （即它是调用 status 的那个）； "suspended"，如果协程在 yield 调用中挂起， 或者尚未开始运行； "normal"，如果协程处于活跃状态但未运行 （即它已恢复了另一个协程）； "dead"，如果协程已完成其主体函数， 或者因错误而停止。

coroutine.wrap (f)

创建一个新协程，主体为 f； f 必须是一个函数。 返回一个函数，每次调用时恢复该协程。 传递给此函数的任何参数 表现得如同传递给 resume 的额外参数。 该函数返回 resume 返回的相同值， 但不包括第一个布尔值。 在出错情况下， 该函数关闭协程并传播错误。

coroutine.yield (···)

挂起调用协程的执行。 传递给 yield 的任何参数 作为额外的结果传递给 resume。

6.4 – 模块

包库提供了在 Lua 中加载模块的基本设施。 它直接在全局环境中导出一个函数： require。 其他所有内容都在表 package 中导出。

require (modname)

加载给定的模块。 函数首先查找 package.loaded 表 以确定 modname 是否已加载。 如果是，则 require 返回存储在 package.loaded[modname] 中的值。 （在此情况下缺少第二个结果 表示此调用不需要加载模块。） 否则，它尝试为该模块找到一个加载器。

要找到加载器， require 由 package.searchers 表引导。 此表中的每一项都是一个搜索函数， 以特定方式搜索模块。 通过更改此表， 我们可以更改 require 查找模块的方式。 以下解释基于 package.searchers 的默认配置。

首先 require 查询 package.preload[modname]。 如果它有值， 这个值（必须是函数）就是加载器。 否则 require 使用存储在 package.path 中的路径搜索 Lua 加载器。 如果这也失败了，它使用存储在 package.cpath 中的路径搜索 C 加载器。 如果这也失败了， 它尝试一体化加载器（参见 package.searchers）。

一旦找到加载器， require 以两个参数调用加载器： modname 和一个额外值， 即加载器数据， 也由搜索器返回。 加载器数据可以是对模块有用的任何值； 对于默认搜索器， 它指示加载器是在哪里找到的。 （例如，如果加载器来自文件， 这个额外值就是文件路径。） 如果加载器返回任何非 nil 值， require 将返回的值赋给 package.loaded[modname]。 如果加载器未返回非 nil 值 且未将任何值赋给 package.loaded[modname]， 则 require 将 true 赋给此条目。 在任何情况下，require 返回 package.loaded[modname] 的最终值。 除该值外，require 还作为第二个结果返回 搜索器返回的加载器数据， 这指示了 require 如何找到该模块。

如果在加载或运行模块时出错， 或者找不到该模块的任何加载器， 则 require 引发错误。

package.config

一个描述包的一些编译时配置的字符串。 此字符串是一系列行：

  • 第一行是目录分隔符字符串。 Windows 上默认为 '\'，其他所有系统上默认为 '/'。
  • 第二行是分隔路径中模板的字符。 默认为 ';'。
  • 第三行是标记模板中替换点的字符串。 默认为 '?'。
  • 第四行是一个字符串，在 Windows 的路径中， 被替换为可执行文件的目录。 默认为 '!'。
  • 第五行是一个标记，用于在构建 luaopen_ 函数名时 忽略其后的所有文本。 默认为 '-'。

package.cpath

一个包含 require 用于 搜索 C 加载器的路径的字符串。

Lua 以与初始化 Lua 路径 package.path 相同的方式 初始化 C 路径 package.cpath， 使用环境变量 LUA_CPATH_5_5， 或环境变量 LUA_CPATH， 或 luaconf.h 中定义的默认路径。

package.loaded

require 使用的表来控制 哪些模块已加载。 当你 require 一个模块 modname 且 package.loaded[modname] 不为 false 时， require 简单地返回存储在那里的值。

此变量只是对实际表的引用； 对此变量的赋值不会更改 require 使用的表。 实际表存储在 C 注册表中（参见 §4.3）， 以字符串 LUA_LOADED_TABLE 为键索引。

package.loadlib (libname, funcname)

将宿主程序与 C 库 libname 动态链接。

如果 funcname 是 "*"， 则它只与库链接， 使库导出的符号 可用于其他动态链接的库。 否则， 它在库内查找函数 funcname 并将该函数作为 C 函数返回。 因此，funcname 必须遵循 lua_CFunction 原型 （参见 lua_CFunction）。

这是一个底层函数。 它完全绕过包和模块系统。 与 require 不同， 它不执行任何路径搜索， 也不自动添加扩展名。 libname 必须是 C 库的完整文件名， 包括必要的路径和扩展名。 funcname 必须是 C 库导出的确切名称 （可能取决于所使用的 C 编译器和链接器）。

此功能不受 ISO C 支持。 因此，loadlib 仅在某些平台上可用： Linux、Windows、Mac OS X、Solaris、BSD， 以及其他支持 dlfcn 标准的 Unix 系统。

此函数本质上是不安全的， 因为它允许 Lua 调用系统中任何可读动态库中的任何函数。 （Lua 调用任何函数时假定该函数 具有正确的原型并遵循正确的协议 （参见 lua_CFunction）。 因此， 调用任意动态库中的任意函数 通常会导致访问违规。）

package.path

一个包含 require 用于 搜索 Lua 加载器的路径的字符串。

启动时，Lua 使用环境变量 LUA_PATH_5_5 的值 或环境变量 LUA_PATH 的值 或 luaconf.h 中定义的默认路径 来初始化此变量， 如果这些环境变量未定义的话。 环境变量值中的 ";;" 被替换为默认路径。

package.preload

一个用于存储特定模块加载器的表 （参见 require）。

此变量只是对实际表的引用； 对此变量的赋值不会更改 require 使用的表。 实际表存储在 C 注册表中（参见 §4.3）， 以字符串 LUA_PRELOAD_TABLE 为键索引。

package.searchers

require 使用的表来控制如何查找模块。

此表中的每个条目都是一个搜索函数。 查找模块时， require 按升序调用这些搜索器中的每一个， 以模块名称（传递给 require 的参数）作为其唯一参数。 如果搜索器找到模块， 它返回另一个函数——模块加载器， 加上一个额外值——加载器数据， 该数据将传递给该加载器并 作为 require 的第二个结果返回。 如果找不到模块， 它返回一个解释原因的字符串 （如果无话可说则返回 nil）。

Lua 用四个搜索函数初始化此表。

第一个搜索器只是在 package.preload 表中查找加载器。

第二个搜索器将加载器作为 Lua 库来查找， 使用存储在 package.path 中的路径。 搜索按照 package.searchpath 函数中描述的方式进行。

第三个搜索器将加载器作为 C 库来查找， 使用 package.cpath 变量给出的路径。 同样， 搜索按照 package.searchpath 函数中描述的方式进行。 例如， 如果 C 路径是字符串 

     "./?.so;./?.dll;/usr/local/?/init.so"

模块 foo 的搜索器 将按顺序尝试打开文件 ./foo.so、./foo.dll 和 /usr/local/foo/init.so。 一旦找到 C 库， 此搜索器首先使用动态链接功能将应用程序与库链接。 然后它尝试在库中找到一个 C 函数 用作加载器。 此 C 函数的名称是字符串 "luaopen_" 连接模块名称的副本（其中每个点替换为下划线）。 此外，如果模块名称包含连字符， 则删除第一个连字符之后（包括该连字符）的后缀。 例如，如果模块名称是 a.b.c-v2.1， 函数名称将是 luaopen_a_b_c。

第四个搜索器尝试一体化加载器。 它在 C 路径中搜索给定模块根名称的库。 例如，require a.b.c 时， 它将搜索 a 的 C 库。 如果找到，它在其中查找子模块的打开函数； 在我们的例子中，那就是 luaopen_a_b_c。 使用此功能，一个包可以将多个 C 子模块打包到一个库中， 每个子模块保留其原始的打开函数。

除第一个（preload）搜索器外，所有搜索器都返回 模块被找到的文件路径作为额外值， 由 package.searchpath 返回。 第一个搜索器始终返回字符串 ":preload:"。

搜索器不应引发错误，也不应在 Lua 中产生副作用。 （它们可能在 C 中产生副作用， 例如通过将应用程序与库链接。）

package.searchpath (name, path [, sep [, rep]])

在给定的 path 中搜索给定的 name。

路径是一个包含一系列由分号分隔的模板的字符串。 对于每个模板， 函数将模板中的每个问号（如果有） 替换为 name 的副本 （其中 sep（默认为点） 的所有出现都替换为 rep （默认为系统的目录分隔符））， 然后尝试打开生成的文件名。

例如，如果路径是字符串 

     "./?.lua;./?.lc;/usr/local/?/init.lua"

搜索名称 foo.a 将按顺序尝试打开文件 ./foo/a.lua、./foo/a.lc 和 /usr/local/foo/a/init.lua。

返回第一个能以读取模式打开的文件的结果名称 （在关闭文件后）， 如果都不成功则返回 fail 加上错误消息。 （此错误消息列出了它尝试打开的所有文件名。）

6.5 – 字符串操作

此库提供字符串操作的通用函数， 如查找和提取子字符串以及模式匹配。 在 Lua 中索引字符串时，第一个字符在位置 1 （不像 C 中从 0 开始）。 索引可以是负数，被解释为从字符串末尾反向索引。 因此，最后一个字符在位置 -1，以此类推。

字符串库在表 string 中提供所有函数。 它还为字符串设置一个元表， 其中 __index 字段指向 string 表。 因此，你可以以面向对象的方式使用字符串函数。 例如，string.byte(s,i) 可以写成 s:byte(i)。

字符串库假定单字节字符编码。

string.byte (s [, i [, j]])

返回字符 s[i]、 s[i+1]、...、s[j] 的内部数值编码。 i 的默认值为 1； j 的默认值为 i。 这些索引按照与函数 string.sub 相同的规则进行修正。

数值编码在不同平台间不一定可移植。

string.char (···)

接收零个或多个整数。 返回一个长度等于参数个数的字符串， 其中每个字符的内部数值编码等于 其对应的参数。

数值编码在不同平台间不一定可移植。

string.dump (function [, strip])

返回一个包含给定函数的二进制表示 （二进制代码块）的字符串， 以便后续对此字符串执行 load 时 返回该函数的副本（但具有新的上值）。 如果 strip 为真值， 二进制表示可能不包含函数的所有调试信息， 以节省空间。

具有上值的函数只保存其上值的数量。 当（重新）加载时， 这些上值会收到全新的实例。 （有关这些上值如何初始化的详细信息， 请参见 load 函数。 你可以使用调试库以适合你需求的方式 序列化和重新加载函数的上值。）

string.find (s, pattern [, init [, plain]])

在字符串 s 中查找 pattern（参见 §6.5.1）的第一次匹配。 如果找到匹配，则 find 返回 s 中 该匹配开始和结束的索引； 否则返回 fail。 第三个可选的数字参数 init 指定 从哪里开始搜索； 其默认值为 1，可以是负数。 第四个可选参数 plain 为 true 时 关闭模式匹配功能， 因此函数执行纯"查找子字符串"操作， pattern 中没有字符被视为特殊字符。

如果模式包含捕获， 则在成功匹配中， 捕获的值也在两个索引之后返回。

string.format (formatstring, ···)

返回其可变数量参数的格式化版本， 遵循其第一个参数（必须是字符串）中的描述。 格式字符串遵循与 ISO C 函数 sprintf 相同的规则。 接受的转换说明符有 A、a、c、d、E、e、f、G、g、 i、o、p、s、u、X、x 和 '%'， 加上非 C 说明符 q。 接受的标志有 '-'、'+'、'#'、 '0' 和 ' '（空格）。 宽度和精度在存在时， 都限制为两位数。

说明符 q 以结果是 Lua 源代码中有效常量的方式 格式化布尔值、nil、数字和字符串。 布尔值和 nil 以明显的方式编写 （true、false、nil）。 浮点数以十六进制编写， 以保留完全精度。 字符串写在双引号之间， 必要时使用转义序列以确保 它可以被 Lua 解释器安全地读回。 例如，调用 

     string.format('%q', 'a string with "quotes" and \n new line')

可能产生字符串： 

     "a string with \"quotes\" and \n
      new line"

此说明符不支持修饰符（标志、宽度、精度）。

转换说明符 A、a、E、e、f、 G 和 g 都期望一个数字作为参数。 说明符 c、d、 i、o、u、X 和 x 期望一个整数。 当 Lua 使用 C89 编译器编译时， 说明符 A 和 a（十六进制浮点数） 不支持修饰符。

说明符 s 期望一个字符串； 如果其参数不是字符串， 则按照与 tostring 相同的规则转换为字符串。 如果说明符有任何修饰符， 相应的字符串参数不应包含嵌入的零。

说明符 p 格式化 lua_topointer 返回的指针。 这为表、userdata、 线程、字符串和函数提供唯一的字符串标识符。 对于其他值（数字、nil、布尔值）， 此说明符生成表示指针 NULL 的字符串。

string.gmatch (s, pattern [, init])

返回一个迭代器函数， 每次调用时返回 pattern（参见 §6.5.1） 在字符串 s 上的下一个捕获。 如果 pattern 未指定捕获， 则每次调用都产生整个匹配。 第三个可选的数字参数 init 指定 从哪里开始搜索； 其默认值为 1，可以是负数。

例如，以下循环 将遍历字符串 s 中的所有单词， 每行打印一个： 

     s = "hello world from Lua"
     for w in string.gmatch(s, "%a+") do
       print(w)
     end

下一个示例将给定字符串中所有 key=value 对 收集到一个表中： 

     t = {}
     s = "from=world, to=Lua"
     for k, v in string.gmatch(s, "(%w+)=(%w+)") do
       t[k] = v
     end

对于此函数，模式开头的脱字符 '^' 不作为锚点工作，因为这会阻止迭代。

string.gsub (s, pattern, repl [, n])

返回 s 的副本， 其中所有（或前 n 个，如果给定）pattern（参见 §6.5.1） 的出现都已替换为由 repl 指定的替换字符串， repl 可以是字符串、表或函数。 gsub 还作为第二个值返回 发生的匹配总数。 名称 gsub 来自全局替换（Global SUBstitution）。

如果 repl 是字符串，则其值用作替换。 字符 % 用作转义字符： repl 中任何形式为 %d 的序列， 其中 d 在 1 到 9 之间， 代表第 d 个捕获的子字符串； 序列 %0 代表整个匹配； 序列 %% 代表单个 %。

如果 repl 是表，则每次匹配时都查询该表， 使用第一个捕获作为键。

如果 repl 是函数，则每次匹配时都调用此函数， 所有捕获的子字符串按顺序作为参数传递。

在任何情况下， 如果模式未指定捕获， 则其行为就好像整个模式都在捕获中一样。

如果表查询或函数调用返回的值 是字符串或数字， 则将其用作替换字符串； 否则，如果为 false 或 nil， 则不进行替换 （即字符串中保留原始匹配）。

以下是一些示例： 

     x = string.gsub("hello world", "(%w+)", "%1 %1")
     -- x="hello hello world world"
     
     x = string.gsub("hello world", "%w+", "%0 %0", 1)
     -- x="hello hello world"
     
     x = string.gsub("hello world from Lua", "(%w+)%s*(%w+)", "%2 %1")
     -- x="world hello Lua from"
     
     x = string.gsub("home = $HOME, user = $USER", "%$(%w+)", os.getenv)
     -- x="home = /home/roberto, user = roberto"
     
     x = string.gsub("4+5 = $return 4+5$", "%$(.-)%$", function (s)
           return load(s)()
         end)
     -- x="4+5 = 9"
     
     local t = {name="lua", version="5.5"}
     x = string.gsub("$name-$version.tar.gz", "%$(%w+)", t)
     -- x="lua-5.5.tar.gz"

string.len (s)

接收一个字符串并返回其长度。 空字符串 "" 的长度为 0。 嵌入的零被计入， 因此 "a\000bc\000" 的长度为 5。

string.lower (s)

接收一个字符串并返回此字符串的副本， 其中所有大写字母都更改为小写。 所有其他字符保持不变。 大写字母的定义取决于当前区域设置。

string.match (s, pattern [, init])

在字符串 s 中查找 pattern（参见 §6.5.1）的第一次匹配。 如果找到，则 match 返回 模式的捕获； 否则返回 fail。 如果 pattern 未指定捕获， 则返回整个匹配。 第三个可选的数字参数 init 指定 从哪里开始搜索； 其默认值为 1，可以是负数。

string.pack (fmt, v1, v2, ···)

返回一个二进制字符串，其中包含值 v1、v2 等， 按照格式字符串 fmt（参见 §6.5.2）以二进制形式（打包）序列化。

string.packsize (fmt)

返回使用给定格式的 string.pack 结果字符串的长度。 格式字符串不能包含变长选项 's' 或 'z'（参见 §6.5.2）。

string.rep (s, n [, sep])

返回一个字符串，它是 n 个 s 字符串副本的连接， 由字符串 sep 分隔。 sep 的默认值为空字符串 （即无分隔符）。 如果 n 不为正数则返回空字符串。

（注意，只需一次调用此函数 就很容易耗尽机器的内存。）

string.reverse (s)

返回一个字符串，它是字符串 s 的反转。

string.sub (s, i [, j])

返回 s 的子字符串， 从 i 开始并延续到 j； i 和 j 可以是负数。 如果 j 不存在，则假定为 -1 （与字符串长度相同）。 特别是， 调用 string.sub(s,1,j) 返回 s 的前缀， 长度为 j， 而 string.sub(s,-i)（对于正数 i） 返回 s 的后缀， 长度为 i。

如果在翻译负索引后， i 小于 1， 则修正为 1。 如果 j 大于字符串长度， 则修正为该长度。 如果在这些修正后， i 大于 j， 函数返回空字符串。

string.unpack (fmt, s [, pos])

返回打包在字符串 s 中的值（参见 string.pack）， 按照格式字符串 fmt（参见 §6.5.2）解包。 可选的 pos 标记 从 s 中开始读取的位置（默认为 1）。 读取值后， 此函数还返回 s 中第一个未读取字节的索引。

string.upper (s)

接收一个字符串并返回此字符串的副本， 其中所有小写字母都更改为大写。 所有其他字符保持不变。 小写字母的定义取决于当前区域设置。

6.5.1 – 模式

Lua 中的模式由普通字符串描述， 由模式匹配函数 string.find、 string.gmatch、 string.gsub 和 string.match 解释为模式。 本节描述这些字符串的语法和含义 （即它们匹配什么）。

字符类：

字符类用于表示一组字符。 描述字符类时允许以下组合：

  • x： （其中 x 不是特殊字符 ^$()%.[]*+-? 之一） 代表字符 x 本身。
  • .：（点）代表所有字符。
  • %a：代表所有字母。
  • %c：代表所有控制字符。
  • %d：代表所有数字。
  • %g：代表除空格外的所有可打印字符。
  • %l：代表所有小写字母。
  • %p：代表所有标点字符。
  • %s：代表所有空白字符。
  • %u：代表所有大写字母。
  • %w：代表所有字母数字字符。
  • %x：代表所有十六进制数字。
  • %x：（其中 x 是任何非字母数字字符） 代表字符 x。 这是转义特殊字符的标准方式。 任何非字母数字字符 （包括所有标点字符，甚至是非特殊的） 都可以在前面加上 '%' 来在模式中代表自身。
  • [set]： 代表由 set 中所有字符的并集组成的类。 字符范围可以通过 按升序用 '-' 分隔范围的端字符来指定。 上面描述的所有 %x 类也可以用作 set 中的组件。 set 中的所有其他字符代表自身。 例如，[%w_]（或 [_%w]） 代表所有字母数字字符加下划线， [0-7] 代表八进制数字， [0-7%l%-] 代表八进制数字加 小写字母加 '-' 字符。

    你可以通过将右方括号放在集合的第一个字符位置 来将其放入集合中。 你可以通过将连字符放在集合的第一个或最后一个字符位置 来将其放入集合中。 （对于这两种情况你也可以使用转义。）

    范围和类之间的交互未定义。 因此，[%a-z] 或 [a-%%] 等模式没有意义。

  • [^set]： 代表 set 的补集， 其中 set 如上所述。

对于所有用单个字母表示的类（%a、%c 等）， 相应的大写字母代表该类的补集。 例如，%S 代表所有非空白字符。

字母、空格和其他字符组的定义 取决于当前区域设置。 特别是，类 [a-z] 可能不等同于 %l。

模式项：

模式项可以是

  • 一个单一字符类， 匹配类中的任何单个字符；
  • 一个单一字符类后跟 '*'， 匹配类中零个或多个字符的序列。 这些重复项将始终匹配最长的可能序列；
  • 一个单一字符类后跟 '+'， 匹配类中一个或多个字符的序列。 这些重复项将始终匹配最长的可能序列；
  • 一个单一字符类后跟 '-'， 也匹配类中零个或多个字符的序列。 与 '*' 不同的是， 这些重复项将始终匹配最短的可能序列；
  • 一个单一字符类后跟 '?'， 匹配类中零个或一个字符。 如果可能，它始终匹配一个；
  • %n，其中 n 在 1 到 9 之间； 此类项匹配等于第 n 个捕获字符串的子字符串 （见下文）；
  • %bxy，其中 x 和 y 是两个不同的字符； 此类项匹配以 x 开头、以 y 结尾， 且 x 和 y 是平衡的字符串。 这意味着，如果从左到右读取字符串， 对 x 计 +1，对 y 计 -1， 则结束的 y 是计数到达 0 时的第一个 y。 例如，项 %b() 匹配具有平衡括号的表达式。
  • %f[set]，一个边界模式； 此类项匹配在任何位置的空字符串， 条件是下一个字符属于 set 且前一个字符不属于 set。 set 的解释如前所述。 主体的开头和结尾被当作字符 '\0' 处理。

模式：

模式是模式项的序列。 模式开头的脱字符 '^' 将匹配锚定在 主体字符串的开头。 模式末尾的 '$' 将匹配锚定在 主体字符串的末尾。 在其他位置， '^' 和 '$' 没有特殊含义，代表自身。

捕获：

模式可以包含括在括号中的子模式； 它们描述捕获。 当匹配成功时， 主体字符串中与捕获匹配的子字符串 被存储（捕获）以供将来使用。 捕获根据其左括号编号。 例如，在模式 "(a*(.)%w(%s*))" 中， 与 "a*(.)%w(%s*)" 匹配的字符串部分 被存储为第一个捕获，因此编号为 1； 与 "." 匹配的字符编号为 2， 与 "%s*" 匹配的部分编号为 3。

作为特殊情况，捕获 () 捕获 当前字符串位置（一个数字）。 例如，如果我们在字符串 "flaaap" 上 应用模式 "()aa()"，将有两个捕获：3 和 5。

多次匹配：

函数 string.gsub 和迭代器 string.gmatch 在主体中匹配给定模式的多次出现。 对于这些函数， 只有当新匹配在前一个匹配结束至少一个字节后结束时， 才被认为是有效的。 换句话说，模式机永远不会在接受另一个匹配后 立即接受空字符串作为匹配。 例如， 考虑以下代码的结果： 

     > string.gsub("abc", "()a*()", print);
     --> 1   2
     --> 3   3
     --> 4   4

第二个和第三个结果来自 Lua 在 'b' 之后 匹配一个空字符串，以及在 'c' 之后再匹配一个。 Lua 不在 'a' 之后匹配空字符串， 因为它会在与前一个匹配相同的位置结束。

6.5.2 – Pack 和 Unpack 的格式字符串

string.pack、 string.packsize 和 string.unpack 的第一个参数是格式字符串， 描述正在创建或读取的结构的布局。

格式字符串是转换选项的序列。 转换选项如下：

  • <：设置小端序
  • >：设置大端序
  • =：设置本机端序
  • ![n]：设置最大对齐为 n （默认为本机对齐）
  • b：有符号字节（char）
  • B：无符号字节（char）
  • h：有符号 short（本机大小）
  • H：无符号 short（本机大小）
  • l：有符号 long（本机大小）
  • L：无符号 long（本机大小）
  • j：一个 lua_Integer
  • J：一个 lua_Unsigned
  • T：一个 size_t（本机大小）
  • i[n]：有符号 int，n 字节 （默认为本机大小）
  • I[n]：无符号 int，n 字节 （默认为本机大小）
  • f：一个 float（本机大小）
  • d：一个 double（本机大小）
  • n：一个 lua_Number
  • cn：固定大小的字符串，n 字节
  • z：以零结尾的字符串
  • s[n]：以长度为前缀的字符串， 长度编码为 n 字节的无符号整数 （默认为 size_t）
  • x：一个字节的填充
  • Xop：一个空项，根据选项 op 对齐 （否则该选项被忽略）
  • ' '：（空格）忽略

（"[n]" 表示可选的整数字面量。） 除填充、空格和配置 （选项 "xX <=>!"）外， 每个选项对应 string.pack 中的一个参数 或 string.unpack 中的一个结果。

对于选项 "!n"、"sn"、"in" 和 "In"， n 可以是 1 到 16 之间的任何整数。 所有整数选项都检查溢出； string.pack 检查给定值是否适合给定大小； string.unpack 检查读取的值是否适合 Lua 整数。 对于无符号选项， Lua 整数也被视为无符号值。

任何格式字符串都以 "!1=" 为前缀开始， 即最大对齐为 1（无对齐）且本机端序。

本机端序假定整个系统 是大端或小端的。 打包函数不会正确模拟混合端序格式的行为。

对齐的工作方式如下： 对于每个选项， 格式在数据开始之前获取额外的填充， 直到数据从一个偏移量开始， 该偏移量是选项大小和最大对齐之间较小值的倍数； 此最小值必须是 2 的幂。 选项 "c" 和 "z" 不对齐； 选项 "s" 遵循其起始整数的对齐方式。

所有填充都由 string.pack 用零填充， 由 string.unpack 忽略。

6.6 – UTF-8 支持

此库提供对 UTF-8 编码的基本支持。 它在表 utf8 中提供所有函数。 此库不提供编码处理以外的任何 Unicode 支持。 任何需要字符含义的操作， 如字符分类，都不在其范围内。

除非另有说明， 所有期望字节位置作为参数的函数 都假定给定位置要么是字节序列的开头， 要么是主体字符串长度加一。 与字符串库一样， 负索引从字符串末尾开始计数。

创建字节序列的函数 接受直到 0x7FFFFFFF 的所有值， 如原始 UTF-8 规范中所定义； 这意味着最多六个字节的字节序列。

解释字节序列的函数只接受 有效序列（格式正确且非超长）。 默认情况下，它们只接受 产生有效 Unicode 码点的字节序列， 拒绝大于 10FFFF 的值和代理对。 布尔参数 lax（如果可用）放宽这些检查， 使得直到 0x7FFFFFFF 的所有值都被接受。 （格式不正确和超长序列仍然被拒绝。）

utf8.char (···)

接收零个或多个整数， 将每个整数转换为对应的 UTF-8 字节序列， 并返回所有这些序列连接而成的字符串。

utf8.charpattern

模式（字符串，不是函数）"[\0-\x7F\xC2-\xFD][\x80-\xBF]*" （参见 §6.5.1）， 假设主体是有效的 UTF-8 字符串， 它恰好匹配一个 UTF-8 字节序列。

utf8.codes (s [, lax])

返回值使得构造 

     for p, c in utf8.codes(s) do body end

将遍历字符串 s 中的所有 UTF-8 字符， 其中 p 是每个字符的位置（以字节为单位），c 是码点。 如果遇到任何无效字节序列，它会引发错误。

utf8.codepoint (s [, i [, j [, lax]]])

返回从字节位置 i 到 j（含）之间开始的 s 中所有字符的码点（作为整数）。 i 的默认值为 1，j 的默认值为 i。 如果遇到任何无效字节序列，它会引发错误。

utf8.len (s [, i [, j [, lax]]])

返回字符串 s 中 位置 i 和 j（含）之间开始的 UTF-8 字符的数量。 i 的默认值为 1，j 的默认值为 -1。 如果发现任何无效字节序列， 返回 fail 加上第一个无效字节的位置。

utf8.offset (s, n [, i])

返回 s 的第 n 个字符的位置 （从字节位置 i 开始计数），作为两个整数： 其编码开始的索引（以字节为单位） 和结束的索引（以字节为单位）。

如果指定的字符正好在 s 的末尾之后， 函数的行为就好像那里有一个 '\0'。 如果指定的字符既不在主体中 也不在其末尾之后， 函数返回 fail。

负数 n 获取位置 i 之前的字符。 i 的默认值在 n 非负时为 1， 否则为 #s + 1， 使得 utf8.offset(s,-n) 获取字符串末尾第 n 个字符的偏移量。

作为特殊情况， 当 n 为 0 时，函数返回包含 s 的第 i 个字节的字符的编码的开始和结束。

此函数假定 s 是有效的 UTF-8 字符串。

6.7 – 表操作

此库提供表操作的通用函数。 它在表 table 中提供所有函数。

请记住，每当操作需要表的长度时， 关于长度运算符的所有注意事项都适用（参见 §3.4.7）。 所有函数都忽略作为参数给出的表中的非数值键。

table.concat (list [, sep [, i [, j]]])

给定一个所有元素都是字符串或数字的列表， 返回字符串 list[i]..sep..list[i+1] ··· sep..list[j]。 sep 的默认值为空字符串， i 的默认值为 1， j 的默认值为 #list。 如果 i 大于 j，返回空字符串。

table.create (nseq [, nrec])

创建一个新的空表，预分配内存。 当你预先知道表将有多少元素时， 此预分配有助于提高性能并节省内存。

参数 nseq 是表将作为序列拥有的元素数量的提示。 可选参数 nrec 是表将拥有的其他元素数量的提示； 其默认值为零。

table.insert (list, [pos,] value)

在 list 的位置 pos 插入元素 value， 将元素 list[pos],list[pos+1],···,list[#list] 向上移动。 pos 的默认值为 #list+1， 因此调用 table.insert(t,x) 将 x 插入到列表 t 的末尾。

table.move (a1, f, e, t [,a2])

将元素从表 a1 移动到表 a2， 执行等同于以下多重赋值的操作： a2[t],··· = a1[f],···,a1[e]。 a2 的默认值为 a1。 目标范围可以与源范围重叠。 要移动的元素数量必须适合一个 Lua 整数。 如果 f 大于 e， 则不移动任何内容。

返回目标表 a2。

table.pack (···)

返回一个新表，所有参数存储在键 1、2 等中， 并有一个字段 "n" 存储参数的总数。 请注意，如果某些参数是 nil， 结果表可能不是序列。

table.remove (list [, pos])

从 list 中删除位置 pos 处的元素， 返回被删除元素的值。 当 pos 是 1 到 #list 之间的整数时， 它将元素 list[pos+1],list[pos+2],···,list[#list] 向下移动， 并删除元素 list[#list]； 索引 pos 也可以在 #list 为 0 时为 0， 或为 #list + 1。

pos 的默认值为 #list， 因此调用 table.remove(l) 删除列表 l 的最后一个元素。

table.sort (list [, comp])

按给定顺序就地对列表元素排序， 从 list[1] 到 list[#list]。 如果给出了 comp， 则它必须是一个接收两个列表元素的函数， 当第一个元素在最终顺序中必须排在第二个元素之前时返回 true， 使得排序后 i <= j 意味着 not comp(list[j],list[i])。 如果未给出 comp， 则使用标准 Lua 运算符 <。

comp 函数必须定义一个一致的顺序； 更正式地说，该函数必须定义一个严格弱序。 （弱序类似于全序， 但可以为比较目的将不同元素等同。）

排序算法不是稳定的： 给定顺序中被认为相等的不同元素 其相对位置可能被排序更改。

table.unpack (list [, i [, j]])

返回给定列表中的元素。 此函数等同于 

     return list[i], list[i+1], ···, list[j]

默认情况下，i 为 1，j 为 #list。

6.8 – 数学函数

此库提供基本数学函数。 它在表 math 中提供所有函数和常量。 注释为 "integer/float" 的函数 对整数参数返回整数结果， 对非整数参数返回浮点结果。 取整函数 math.ceil、math.floor 和 math.modf 在结果适合整数范围时返回整数， 否则返回浮点数。

math.abs (x)

返回 x 和 -x 之间的最大值。（整数/浮点数）

math.acos (x)

返回 x 的反余弦值（以弧度为单位）。

math.asin (x)

返回 x 的反正弦值（以弧度为单位）。

math.atan (y [, x])

返回 y/x 的反正切值（以弧度为单位）， 使用两个参数的符号来确定结果的象限。 它还正确处理 x 为零的情况。

x 的默认值为 1， 因此调用 math.atan(y) 返回 y 的反正切值。

math.ceil (x)

返回大于或等于 x 的最小整数值。

math.cos (x)

返回 x 的余弦值（假定以弧度为单位）。

math.deg (x)

将角度 x 从弧度转换为度。

math.exp (x)

返回值 ex （其中 e 是自然对数的底数）。

math.floor (x)

返回小于或等于 x 的最大整数值。

math.fmod (x, y)

返回 x 除以 y 的余数， 其中商向零取整。（整数/浮点数）

math.frexp (x)

返回两个数字 m 和 e，使得 x = m2e， 其中 e 是整数。 当 x 为零、NaN、+inf 或 -inf 时， m 等于 x； 否则，m 的绝对值在范围 [0.5, 1) 内。

math.huge

浮点值 HUGE_VAL， 一个大于任何其他数值的值。

math.ldexp (m, e)

返回 m2e，其中 e 是整数。

math.log (x [, base])

返回 x 以给定底数的对数。 base 的默认值为 e （因此函数返回 x 的自然对数）。

math.max (x, ···)

返回具有最大值的参数， 按照 Lua 运算符 < 确定。

math.maxinteger

一个具有整数最大值的整数。

math.min (x, ···)

返回具有最小值的参数， 按照 Lua 运算符 < 确定。

math.mininteger

一个具有整数最小值的整数。

math.modf (x)

返回 x 的整数部分和 x 的小数部分。 其第二个结果始终是浮点数。

math.pi

π 的值。

math.rad (x)

将角度 x 从度转换为弧度。

math.random ([m [, n]])

不带参数调用时， 返回在范围 [0, 1) 内均匀分布的伪随机浮点数。 以两个整数 m 和 n 调用时， math.random 返回在范围 [m, n] 内均匀分布的伪随机整数。 对于正数 n，调用 math.random(n) 等同于 math.random(1,n)。 调用 math.random(0) 产生一个 所有位（伪）随机的整数。

此函数使用 xoshiro256** 算法来生成 伪随机 64 位整数， 这些整数是参数为 0 的调用的结果。 其他结果（范围和浮点数） 从这些整数中无偏提取。

Lua 使用等同于不带参数调用 math.randomseed 的方式 初始化其伪随机生成器， 使得 math.random 应该在程序每次运行时 生成不同的结果序列。

math.randomseed ([x [, y]])

至少带一个参数调用时， 整数参数 x 和 y 被连接成一个种子， 用于重新初始化伪随机生成器； 相等的种子产生相等的数字序列。 y 的默认值为零。

不带参数调用时， Lua 生成一个种子， 以弱随机性尝试来确保一定的随机性。

此函数返回实际使用的两个种子组件， 以便再次设置它们时可以重复该序列。

为了确保初始状态所需的随机性水平 （或者相反，为了获得确定性序列， 例如在调试程序时）， 你应该使用显式参数调用 math.randomseed。

math.sin (x)

返回 x 的正弦值（假定以弧度为单位）。

math.sqrt (x)

返回 x 的平方根。 （你也可以使用表达式 x^0.5 来计算此值。）

math.tan (x)

返回 x 的正切值（假定以弧度为单位）。

math.tointeger (x)

如果值 x 可以转换为整数， 则返回该整数。 否则返回 fail。

math.type (x)

如果 x 是整数则返回 "integer"， 如果是浮点数则返回 "float"， 如果 x 不是数字则返回 fail。

math.ult (m, n)

返回一个布尔值， 当且仅当整数 m 作为无符号整数比较时小于整数 n 时为 true。

6.9 – 输入和输出设施

I/O 库提供两种不同的文件操作风格。 第一种使用隐式文件句柄； 即，有操作来设置默认输入文件和默认输出文件， 所有输入/输出操作都在这些默认文件上进行。 第二种风格使用显式文件句柄。

使用隐式文件句柄时， 所有操作由表 io 提供。 使用显式文件句柄时， 操作 io.open 返回一个文件句柄， 然后所有操作作为文件句柄的方法提供。

文件句柄的元表提供 __gc 和 __close 元方法， 在调用时尝试关闭文件。

表 io 还提供三个预定义的文件句柄， 具有来自 C 的通常含义： io.stdin、io.stdout 和 io.stderr。 I/O 库从不关闭这些文件。

除非另有说明， 所有 I/O 函数在失败时返回 fail， 加上错误消息作为第二个结果， 系统相关的错误代码作为第三个结果， 成功时返回某个非假值。 在非 POSIX 系统上， 出错时计算错误消息和错误代码 可能不是线程安全的， 因为它们依赖于全局 C 变量 errno。

io.close ([file])

等同于 file:close()。 不带 file 时，关闭默认输出文件。

io.flush ()

等同于 io.output():flush()。

io.input ([file])

以文件名调用时，它打开指定文件（以文本模式）， 并将其句柄设置为默认输入文件。 以文件句柄调用时， 它只是将此文件句柄设置为默认输入文件。 不带参数调用时， 返回当前默认输入文件。

出错时此函数引发错误， 而不是返回错误代码。

io.lines ([filename, ···])

以读取模式打开给定的文件名， 并返回一个迭代器函数， 其行为类似于 file:lines(···) 在打开的文件上。 当迭代器函数无法读取任何值时， 它自动关闭文件。 除迭代器函数外， io.lines 还返回三个其他值： 两个 nil 值作为占位符， 加上创建的文件句柄。 因此，当在通用 for 循环中使用时， 即使循环被错误或 break 中断，文件也会被关闭。

调用 io.lines()（不带文件名）等同于 io.input():lines("l")； 即，它遍历默认输入文件的行。 在这种情况下，迭代器在循环结束时不关闭文件。

打开文件出错时， 此函数引发错误， 而不是返回错误代码。

io.open (filename [, mode])

此函数打开一个文件， 以字符串 mode 中指定的模式打开。 成功时返回一个新的文件句柄。

mode 字符串可以是以下任何一种：

  • "r"：读取模式（默认）；
  • "w"：写入模式；
  • "a"：追加模式；
  • "r+"：更新模式，所有先前数据被保留；
  • "w+"：更新模式，所有先前数据被擦除；
  • "a+"：追加更新模式，先前数据被保留， 写入只允许在文件末尾。

mode 字符串也可以在末尾有一个 'b'， 在某些系统上需要它来以二进制模式打开文件。

io.output ([file])

类似于 io.input，但在默认输出文件上操作。

io.popen (prog [, mode])

此函数是系统相关的，并非在所有平台上都可用。

在独立进程中启动程序 prog 并返回一个文件句柄， 你可以用它来从此程序读取数据 （如果 mode 为 "r"，默认值） 或向此程序写入数据 （如果 mode 为 "w"）。

io.read (···)

等同于 io.input():read(···).

io.tmpfile ()

成功时返回临时文件的句柄。 此文件以更新模式打开， 在程序结束时自动删除。

io.type (obj)

检查 obj 是否是有效的文件句柄。 如果 obj 是打开的文件句柄则返回字符串 "file"， 如果 obj 是关闭的文件句柄则返回 "closed file"， 如果 obj 不是文件句柄则返回 fail。

io.write (···)

等同于 io.output():write(···).

file:close ()

关闭 file。 请注意，文件在其句柄被垃圾回收时会自动关闭， 但这需要不可预测的时间。

关闭使用 io.popen 创建的文件句柄时， file:close 返回与 os.execute 相同的值。

file:flush ()

将所有写入的数据保存到 file。

file:lines (···)

返回一个迭代器函数， 每次调用时按照给定的格式读取文件。 未给定格式时， 使用 "l" 作为默认值。 例如，构造 

     for c in file:lines(1) do body end

将遍历文件的所有字符， 从当前位置开始。 与 io.lines 不同，此函数在循环结束时不关闭文件。

file:read (···)

读取文件 file， 根据给定的格式指定读取内容。 对于每种格式， 函数返回包含读取字符的字符串或数字， 如果无法使用指定格式读取数据则返回 fail。 （在后一种情况下， 函数不会读取后续格式。） 不带参数调用时， 它使用读取下一行的默认格式 （见下文）。

可用的格式有

  • "n"： 读取一个数字并作为浮点数或整数返回， 遵循 Lua 的词法约定。 （数字可以有前导空白和符号。） 此格式始终读取作为数字有效前缀的最长输入序列； 如果该前缀不构成有效数字 （例如空字符串、"0x" 或 "3.4e-"） 或太长（超过 200 个字符）， 则被丢弃，格式返回 fail。
  • "a"： 从当前位置开始读取整个文件。 到达文件末尾时返回空字符串； 此格式永远不会失败。
  • "l"： 读取下一行，跳过行尾， 到达文件末尾时返回 fail。 这是默认格式。
  • "L"： 读取下一行，保留行尾字符（如果有）， 到达文件末尾时返回 fail。
  • number： 读取最多该数字字节数的字符串， 到达文件末尾时返回 fail。 如果 number 为零， 则不读取任何内容，返回空字符串， 到达文件末尾时返回 fail。

格式 "l" 和 "L" 应仅用于文本文件。

file:seek ([whence [, offset]])

设置和获取文件位置， 从文件开头测量， 到由 offset 加上由字符串 whence 指定的基点给出的位置， 如下所示：

  • "set"：基点为位置 0（文件开头）；
  • "cur"：基点为当前位置；
  • "end"：基点为文件末尾；

成功时，seek 返回最终文件位置， 从文件开头以字节为单位测量。 如果 seek 失败，返回 fail， 加上描述错误的字符串。

whence 的默认值为 "cur"， offset 的默认值为 0。 因此，调用 file:seek() 返回当前文件位置，不更改它； 调用 file:seek("set") 将位置设置为文件开头（并返回 0）； 调用 file:seek("end") 将位置设置为文件末尾，并返回其大小。

file:setvbuf (mode [, size])

设置文件的缓冲模式。 有三种可用模式：

  • "no"：无缓冲。
  • "full"：完全缓冲。
  • "line"：行缓冲。

对于后两种情况， size 是缓冲区大小的提示，以字节为单位。 默认值为适当的大小。

每种模式的具体行为是不可移植的； 有关更多详细信息，请查看您平台上底层的 ISO C 函数 setvbuf。

file:write (···)

将每个参数的值写入 file。 参数必须是字符串或数字。

成功时，此函数返回 file。 否则返回四个值： fail、错误消息、错误代码和能够写入的字节数。

6.10 – 操作系统设施

此库通过表 os 实现。

os.clock ()

返回程序使用的 CPU 时间的近似秒数， 由底层的 ISO C 函数 clock 返回。

os.date ([format [, time]])

返回一个包含日期和时间的字符串或表， 按照给定字符串 format 格式化。

如果存在 time 参数， 这是要格式化的时间 （有关此值的描述，请参见 os.time 函数）。 否则，date 格式化当前时间。

如果 format 以 '!' 开头， 则日期以协调世界时（UTC）格式化。 在此可选字符之后， 如果 format 是字符串 "*t"， 则 date 返回一个包含以下字段的表： year、month（1–12）、day（1–31）、 hour（0–23）、min（0–59）、 sec（0–61，由于闰秒）、 wday（星期几，1–7，星期日为 1）、 yday（一年中的第几天，1–366）， 和 isdst（夏令时标志，布尔值）。 如果信息不可用， 最后一个字段可能不存在。

如果 format 不是 "*t"， 则 date 将日期作为字符串返回， 按照与 ISO C 函数 strftime 相同的规则格式化。

如果 format 不存在，默认为 "%c"， 使用当前区域设置提供人类可读的日期和时间表示。

在非 POSIX 系统上， 此函数可能不是线程安全的， 因为它依赖于 C 函数 gmtime 和 C 函数 localtime。

os.difftime (t2, t1)

返回从时间 t1 到时间 t2 的差（以秒为单位） （其中时间是 os.time 返回的值）。 在 POSIX、Windows 和某些其他系统上， 此值恰好为 t2-t1。

os.execute ([command])

此函数等同于 ISO C 函数 system。 它将 command 传递给操作系统 shell 执行。 其第一个结果在命令成功终止时为 true， 否则为 fail。 在此第一个结果之后， 函数返回一个字符串加一个数字， 如下所示：

  • "exit"：命令正常终止； 后面的数字是命令的退出状态。
  • "signal"：命令被信号终止； 后面的数字是终止命令的信号。

不带 command 调用时， os.execute 返回一个布尔值，如果 shell 可用则为 true。

os.exit ([code [, close]])

调用 ISO C 函数 exit 来终止宿主程序。 如果 code 为 true， 返回的状态为 EXIT_SUCCESS； 如果 code 为 false， 返回的状态为 EXIT_FAILURE； 如果 code 是数字， 返回的状态就是该数字。 code 的默认值为 true。

如果可选的第二个参数 close 为 true， 函数在退出前关闭 Lua 状态（参见 lua_close）。

os.getenv (varname)

返回进程环境变量 varname 的值， 如果变量未定义则返回 fail。

os.remove (filename)

删除具有给定名称的文件（或在 POSIX 系统上为空目录）。 如果此函数失败，返回 fail， 加上描述错误的字符串和错误代码。 否则返回 true。

os.rename (oldname, newname)

将名为 oldname 的文件或目录重命名为 newname。 如果此函数失败，返回 fail， 加上描述错误的字符串和错误代码。 否则返回 true。

os.setlocale (locale [, category])

设置程序的当前区域设置。 locale 是一个系统相关的字符串，指定区域设置； category 是一个可选字符串，描述要更改的类别： "all"、"collate"、"ctype"、 "monetary"、"numeric" 或 "time"； 默认类别为 "all"。 函数返回新区域设置的名称， 如果请求无法满足则返回 fail。

如果 locale 为空字符串， 当前区域设置被设置为实现定义的本机区域设置。 如果 locale 是字符串 "C"， 当前区域设置被设置为标准 C 区域设置。

以 nil 作为第一个参数调用时， 此函数只返回给定类别的当前区域设置名称。

此函数可能不是线程安全的， 因为它依赖于 C 函数 setlocale。

os.time ([table])

不带参数调用时返回当前本地时间， 或返回一个表示给定表指定的本地日期和时间的时间值。 此表必须有字段 year、month 和 day， 可以有字段 hour（默认为 12）、 min（默认为 0）、 sec（默认为 0）， 和 isdst（默认为 nil）。 其他字段被忽略。 有关这些字段的描述，请参见 os.date 函数。

调用函数时， 这些字段中的值不需要在其有效范围内。 例如，如果 sec 为 -10， 它表示在其他字段指定的时间之前 10 秒； 如果 hour 为 1000， 它表示在其他字段指定的时间之后 1000 小时。

返回的值是一个数字，其含义取决于您的系统。 在 POSIX、Windows 和某些其他系统上， 此数字计算自某个给定开始时间（"纪元"）以来的秒数。 在其他系统中，含义未指定， time 返回的数字只能作为 os.date 和 os.difftime 的参数使用。

以表调用时， os.time 还规范化 os.date 函数中记录的所有字段， 使它们表示与调用前相同的时间，但值在其有效范围内。

os.tmpname ()

返回一个可以用作临时文件的文件名字符串。 文件必须在使用前显式打开， 不再需要时显式删除。

在 POSIX 系统上， 此函数还创建具有该名称的文件， 以避免安全风险。 （其他人可能在获取名称和创建文件之间的时间内 以错误的权限创建文件。） 你仍然需要打开文件来使用它 并删除它（即使你不使用它）。

如果可能， 你可能更喜欢使用 io.tmpfile， 它在程序结束时自动删除文件。

6.11 – 调试库

此库为 Lua 程序提供 调试接口（§4.7）的功能。 使用此库时应谨慎。 其多个函数 违反了关于 Lua 代码的基本假设 （例如，函数的局部变量 不能从外部访问； userdata 元表不能被 Lua 代码更改； Lua 程序不会崩溃） 因此可能危及本来安全的代码。 此外，此库中的某些函数可能很慢。

此库中的所有函数都在 debug 表中提供。 所有在线程上操作的函数 都有一个可选的第一个参数， 即要操作的线程。 默认值始终是当前线程。

debug.debug ()

进入与用户的交互模式， 运行用户输入的每个字符串。 使用简单命令和其他调试设施， 用户可以检查全局和局部变量， 更改它们的值，计算表达式等等。 仅包含单词 cont 的行结束此函数， 使调用者继续其执行。

请注意 debug.debug 的命令在词法上不嵌套在任何函数中， 因此无法直接访问局部变量。

debug.gethook ([thread])

以三个值返回线程的当前钩子设置： 当前钩子函数、当前钩子掩码和当前钩子计数， 由 debug.sethook 函数设置。

如果没有活动钩子则返回 fail。

debug.getinfo ([thread,] f [, what])

返回一个包含函数信息的表。 你可以直接给出函数， 也可以给出一个数字作为 f 的值， 这意味着在给定线程调用栈的 f 级别运行的函数： 级别 0 是当前函数（getinfo 本身）； 级别 1 是调用 getinfo 的函数 （尾调用除外，它们不计入栈中）； 以此类推。 如果 f 是大于活动函数数量的数字， 则 getinfo 返回 fail。

返回的表可以包含 lua_getinfo 返回的所有字段， 字符串 what 描述要填充哪些字段。 what 的默认值是获取所有可用信息， 除有效行表外。 选项 'f' 添加一个名为 func 的字段，包含函数本身。 选项 'L' 添加一个名为 activelines 的字段， 包含有效行表， 前提是该函数是 Lua 函数。 如果函数没有调试信息， 表为空。

例如，表达式 debug.getinfo(1,"n").name 返回 当前函数的名称（如果可以找到合理的名称）， 表达式 debug.getinfo(print) 返回一个包含 print 函数所有可用信息的表。

debug.getlocal ([thread,] f, local)

此函数返回栈上 f 级别函数的索引为 local 的局部变量的名称和值。 此函数不仅访问显式局部变量， 还访问参数和临时值。

第一个参数或局部变量的索引为 1，以此类推， 按照它们在代码中声明的顺序， 仅计算在函数当前作用域中活跃的变量。 编译时常量可能不会出现在此列表中， 如果它们已被编译器优化掉。 负索引引用可变参数； -1 是第一个可变参数。 这些负索引仅在可变参数表已被优化掉时可用； 否则，可变参数在可变参数表中可用。

如果没有具有给定索引的变量， 函数返回 fail， 当以超出范围的级别调用时引发错误。 （你可以调用 debug.getinfo 来检查级别是否有效。）

以 '('（左括号）开头的变量名 表示没有已知名称的变量 （内部变量如循环控制变量， 以及没有调试信息保存的代码块中的变量）。

参数 f 也可以是函数。 在这种情况下，getlocal 只返回函数参数的名称。

debug.getmetatable (value)

返回给定 value 的元表， 如果没有元表则返回 nil。

debug.getregistry ()

返回注册表（参见 §4.3）。

debug.getupvalue (f, up)

此函数返回函数 f 的索引为 up 的上值的名称和值。 如果没有具有给定索引的上值， 函数返回 fail。

（对于 Lua 函数， 上值是函数使用的外部局部变量， 因此包含在其闭包中。）

对于 C 函数，此函数使用空字符串 "" 作为所有上值的名称。

变量名 '?'（问号）表示没有已知名称的变量 （没有调试信息保存的代码块中的变量）。

debug.getuservalue (u, n)

返回与 userdata u 关联的第 n 个用户值， 加上一个布尔值， 如果 userdata 没有该值则为 false。

debug.sethook ([thread,] hook, mask [, count])

将给定函数设置为调试钩子。 字符串 mask 和数字 count 描述 何时调用钩子。 字符串掩码可以包含以下字符的任意组合， 具有给定含义：

  • 'c'：每次 Lua 调用函数时调用钩子；
  • 'r'：每次 Lua 从函数返回时调用钩子；
  • 'l'：每次 Lua 进入代码的新行时调用钩子。

此外， 当 count 不为零时， 钩子也在每 count 条指令后调用。

不带参数调用时， debug.sethook 关闭钩子。

当钩子被调用时，其第一个参数是描述触发调用的事件的字符串： "call"、"tail call"、"return"、 "line" 和 "count"。 对于行事件， 钩子还获得新行号作为其第二个参数。 在钩子内部， 你可以以级别 2 调用 getinfo 来获取有关运行函数的更多信息。 （级别 0 是 getinfo 函数， 级别 1 是钩子函数。）

debug.setlocal ([thread,] level, local, value)

此函数将值 value 赋给栈上 level 级别函数的索引为 local 的局部变量。 如果没有具有给定索引的局部变量， 函数返回 fail， 当以超出范围的 level 调用时引发错误。 （你可以调用 getinfo 来检查级别是否有效。） 否则返回局部变量的名称。

有关变量索引和名称的更多信息， 请参见 debug.getlocal。

debug.setmetatable (value, table)

将给定 value 的元表设置为给定的 table （可以是 nil）。 返回 value。

debug.setupvalue (f, up, value)

此函数将值 value 赋给函数 f 的索引为 up 的上值。 如果没有具有给定索引的上值， 函数返回 fail。 否则返回上值的名称。

有关上值的更多信息， 请参见 debug.getupvalue。

debug.setuservalue (udata, value, n)

将给定的 value 设置为 与给定 udata 关联的第 n 个用户值。 udata 必须是完整的 userdata。

返回 udata， 如果 userdata 没有该值则返回 fail。

debug.traceback ([thread,] [message [, level]])

如果 message 存在但既不是字符串也不是 nil， 此函数不进一步处理直接返回 message。 否则， 它返回一个包含调用栈回溯的字符串。 可选的 message 字符串附加在回溯的开头。 可选的 level 数字指定从哪个级别开始回溯 （默认为 1，即调用 traceback 的函数）。

debug.upvalueid (f, n)

返回给定函数的编号为 n 的上值的唯一标识符 （作为轻量级 userdata）。

这些唯一标识符允许程序检查不同的闭包是否共享上值。 共享上值的 Lua 闭包 （即访问相同外部局部变量的闭包） 将对这些上值索引返回相同的 id。

debug.upvaluejoin (f1, n1, f2, n2)

使 Lua 闭包 f1 的第 n1 个上值 引用 Lua 闭包 f2 的第 n2 个上值。

7 – 独立的 Lua 解释器

尽管 Lua 被设计为一种扩展语言，用于嵌入到宿主 C 程序中，但它也经常被用作一种独立的语言。标准发行版中提供了一个独立的 Lua 解释器，简称 lua。该独立解释器包含所有标准库。其用法如下： 

     lua [options] [script [args]]

各选项说明如下：

  • -e stat：执行字符串 stat；
  • -i：在运行 script 之后进入交互模式；
  • -l mod：执行 "require" 加载 mod，并将结果赋值给全局变量 mod；
  • -l g=mod：执行 "require" 加载 mod，并将结果赋值给全局变量 g；
  • -v：打印版本信息；
  • -E：忽略环境变量；
  • -W：开启警告；
  • --：停止处理选项；
  • -：将 stdin 作为文件执行，并停止处理选项。

处理完选项后，lua 会运行给定的 script。当不带参数调用时，如果标准输入（stdin）是终端，lua 的行为等同于 lua -v -i，否则等同于 lua -。

当不带 -E 选项调用时，解释器会在运行任何参数之前检查环境变量 LUA_INIT_5_5（如果版本化名称未定义，则检查 LUA_INIT）。如果变量内容的格式为 @filename，则 lua 会执行该文件。否则，lua 会直接执行该字符串。

当带 -E 选项调用时，Lua 不会读取任何环境变量。特别是，package.path 和 package.cpath 的值会使用 luaconf.h 中定义的默认路径。为了通知各库该选项已启用，独立解释器会将注册表中的 "LUA_NOENV" 字段设置为 true。其他库也可以读取此字段以达到同样的目的。

选项 -e、-l 和 -W 按照它们出现的顺序依次处理。例如，如下调用 

     $ lua -e 'a=1' -llib1 script.lua

会首先将 a 设为 1，然后加载库 lib1，最后以无参数的方式运行文件 script.lua。（其中 $ 是 shell 提示符，你的提示符可能不同。）

在运行任何代码之前，lua 会将所有命令行参数收集到一个名为 arg 的全局表中。脚本名称位于索引 0，脚本名称之后的第一个参数位于索引 1，以此类推。脚本名称之前的参数（即解释器名称及其选项）位于负索引。例如，在如下调用中 

     $ lua -la b.lua t1 t2

该表的内容如下： 

     arg = { [-2] = "lua", [-1] = "-la",
             [0] = "b.lua",
             [1] = "t1", [2] = "t2" }

如果调用中没有脚本，则解释器名称位于索引 0，后面紧跟其他参数。例如，如下调用 

     $ lua -e "print(arg[1])"

将打印 "-e"。如果有脚本，则脚本会使用参数 arg[1]、···、arg[#arg] 来调用。与 Lua 中所有代码块一样，脚本被编译为一个可变参数函数。

在交互模式下，Lua 反复提示并等待输入一行。读入一行后，Lua 首先尝试将该行解释为一个表达式。如果成功，则打印其值。否则，将该行解释为一个代码块。如果你编写了一个不完整的代码块，解释器会通过显示不同的提示符来等待其补全。

请注意，由于每一行完整的输入都会作为新的代码块来读取，局部变量不会在行之间有效。为了避免混淆，如果一行以保留字 local 开头，解释器会给出警告： 

     > x = 20      -- global 'x'
     > local x = 10; print(x)
         --> warning: locals do not survive across lines in interactive mode
         --> 10
     > print(x)     -- back to global 'x'
         --> 20
     > do       -- incomplete chunk
     >> local x = 10; print(x)    -- '>>' prompts for line completion
     >> print(x)
     >> end     -- chunk completed
        --> 10
        --> 10

如果全局变量 _PROMPT 包含一个字符串，则其值被用作提示符。类似地，如果全局变量 _PROMPT2 包含一个字符串，则其值被用作二级提示符（在不完整的语句期间使用）。

当脚本中出现未受保护的错误时，解释器会将错误信息输出到标准错误流。如果错误对象不是字符串，但具有 __tostring 元方法，则解释器会调用该元方法来生成最终的错误消息。否则，解释器会将错误对象转换为字符串，并在其后附加一个栈回溯信息。当警告功能开启时，警告信息会直接输出到标准错误输出。

正常结束时，解释器会关闭其主 Lua 状态（参见 lua_close）。脚本可以通过调用 os.exit 来终止，从而避免这一步骤。

为了允许在 Unix 系统中将 Lua 用作脚本解释器，如果文件块的第一行以 # 开头，Lua 会跳过该行。因此，可以使用 chmod +x 和 #! 形式将 Lua 脚本变为可执行程序，例如： 

     #!/usr/local/bin/lua

当然，Lua 解释器的位置在你的机器上可能不同。如果 lua 在你的 PATH 中，则 

     #!/usr/bin/env lua

是更具可移植性的解决方案。

8 – 与前一版本的不兼容性

这里列出了将程序从 Lua 5.4 迁移到 Lua 5.5 时可能遇到的不兼容性。

你可以通过使用适当的选项编译 Lua（参见 luaconf.h 文件）来避免一些不兼容性。然而，所有这些兼容性选项在未来将会被移除。更常见的情况是，当这些兼容性选项被移除时才会出现兼容性问题。因此，只要有机会，你就应该尝试使用关闭了所有兼容性选项的 Lua 版本来测试你的代码。这将有助于向更新版本的 Lua 过渡。

Lua 版本总是可能以不涉及程序源代码更改的方式更改 C API，例如常量的数值或函数作为宏的实现方式。因此，你永远不应假设不同 Lua 版本之间的二进制文件是兼容的。使用新版本时，应始终重新编译使用 Lua API 的客户端。

类似地，Lua 版本总是可能更改预编译块的内部表示；预编译块在不同 Lua 版本之间是不兼容的。

官方发行版中的标准路径可能会在不同版本之间发生变化。

8.1 – 语言方面的不兼容性

  • 单词 global 是一个保留字。不要将其用作普通名称。
  • for 循环中的控制变量是只读的。如果你需要修改它，请在循环体中声明一个同名的局部变量。
  • __call 元方法的调用链最多只能包含 15 个对象。
  • 在错误中，作为错误对象的 nil 会被替换为一个字符串消息。

8.2 – 库方面的不兼容性

  • 垃圾回收的参数不再使用 "incremental" 和 "generational" 选项来设置；取而代之的是一个新的 "param" 选项。此外，参数本身也有一些变化。

8.3 – API 方面的不兼容性

  • 在 lua_call 及相关函数中，所需结果数量（nresults）的最大值为 250。如果你确实需要更大的值，请使用 LUA_MULTRET，然后调整栈的大小。此前，此限制没有明确规定。
  • lua_newstate 新增了第三个参数，用于字符串哈希的种子值。
  • 函数 lua_resetthread 已被弃用；它的功能等同于 lua_closethread，且 from 参数为 NULL。
  • 函数 lua_setcstacklimit 已被弃用。对其的调用可以直接移除。
  • 函数 lua_dump 改变了在调用写入函数期间保持栈的方式。（这在之前的版本中没有明确规定。）此外，它会额外调用写入函数一次，以标志转储的结束。
  • 垃圾回收的参数不再使用 LUA_GCINC 和 LUA_GCGEN 选项来设置；取而代之的是一个新的 LUA_GCPARAM 选项。此外，参数本身也有一些变化。
  • 函数 lua_pushvfstring 现在会报告错误，而不是抛出错误。

9 – Lua 的完整语法

以下是 Lua 的扩展 BNF 完整语法。与扩展 BNF 的惯例一致，{A} 表示零个或多个 A，[A] 表示可选的 A。（关于运算符优先级，参见 §3.4.8；关于终结符 Name、Numeral 和 LiteralString 的描述，参见 §3.1。） 



	chunk ::= block

	block ::= {stat} [retstat]

	stat ::=  ‘;’ | 
		 varlist ‘=’ explist | 
		 functioncall | 
		 label | 
		 break | 
		 goto Name | 
		 do block end | 
		 while exp do block end | 
		 repeat block until exp | 
		 if exp then block {elseif exp then block} [else block] end | 
		 for Name ‘=’ exp ‘,’ exp [‘,’ exp] do block end | 
		 for namelist in explist do block end | 
		 function funcname funcbody | 
		 local function Name funcbody | 
		 global function Name funcbody | 
		 local attnamelist [‘=’ explist] | 
		 global attnamelist | 
		 global [attrib] ‘*’ 

	attnamelist ::=  [attrib] Name [attrib] {‘,’ Name [attrib]}

	attrib ::= ‘<’ Name ‘>’

	retstat ::= return [explist] [‘;’]

	label ::= ‘::’ Name ‘::’

	funcname ::= Name {‘.’ Name} [‘:’ Name]

	varlist ::= var {‘,’ var}

	var ::=  Name | prefixexp ‘[’ exp ‘]’ | prefixexp ‘.’ Name 

	namelist ::= Name {‘,’ Name}

	explist ::= exp {‘,’ exp}

	exp ::=  nil | false | true | Numeral | LiteralString | ‘...’ | functiondef | 
		 prefixexp | tableconstructor | exp binop exp | unop exp 

	prefixexp ::= var | functioncall | ‘(’ exp ‘)’

	functioncall ::=  prefixexp args | prefixexp ‘:’ Name args 

	args ::=  ‘(’ [explist] ‘)’ | tableconstructor | LiteralString 

	functiondef ::= function funcbody

	funcbody ::= ‘(’ [parlist] ‘)’ block end

	parlist ::= namelist [‘,’ varargparam] | varargparam

	varargparam ::= ‘...’ [Name]

	tableconstructor ::= ‘{’ [fieldlist] ‘}’

	fieldlist ::= field {fieldsep field} [fieldsep]

	field ::= ‘[’ exp ‘]’ ‘=’ exp | Name ‘=’ exp | exp

	fieldsep ::= ‘,’ | ‘;’

	binop ::=  ‘+’ | ‘-’ | ‘*’ | ‘/’ | ‘//’ | ‘^’ | ‘%’ | 
		 ‘&’ | ‘~’ | ‘|’ | ‘>>’ | ‘<<’ | ‘..’ | 
		 ‘<’ | ‘<=’ | ‘>’ | ‘>=’ | ‘==’ | ‘~=’ | 
		 and | or

	unop ::= ‘-’ | not | ‘#’ | ‘~’
