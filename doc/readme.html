欢迎使用 Lua 5.5

关于 · 安装 · 变更 · 许可证 · 参考手册

关于 Lua

Lua 是一种功能强大、高效、轻量级的嵌入式脚本语言， 由 团队 在 PUC-Rio （巴西里约热内卢天主教大学）开发。 Lua 是 自由软件， 被全球 众多产品和项目使用。

Lua 的 官方网站 提供关于 Lua 的完整信息， 包括 概要介绍、 入门指南 以及 最新文档， 特别是 参考手册， 它可能与本发行版中的 本地副本略有不同。

安装 Lua

Lua 以 源代码形式发行。 你需要在使用前构建它。 构建 Lua 应该很简单， 因为 Lua 是用纯 ISO C 实现的， 可以在所有已知拥有 ISO C 编译器的平台上不加修改地编译。 Lua 也可以不加修改地作为 C++ 编译。 下面给出的构建 Lua 的说明适用于类 Unix 平台， 如 Linux 和 macOS。 另请参见 其他系统的说明和 自定义选项。

如果你没有时间或意愿自己编译 Lua， 可以从 LuaBinaries获取二进制文件。

构建 Lua

在大多数常见的类 Unix 平台上，只需执行 "make"。 以下是详细步骤。

  1. 打开终端窗口并移动到 名为 lua-5.5.0 的顶级目录。 那里的 Makefile 控制构建过程和安装过程。

  2. 执行 "make"。Makefile 将猜测你的平台并为它构建 Lua。

  3. 如果猜测失败，执行 "make help" 查看你的平台是否列出。 目前支持的平台有：

    guess aix bsd c89 freebsd generic ios linux macosx mingw posix solaris

    如果你的平台列出了，只需执行 "make xxx"，其中 xxx 是你的平台名称。

    如果你的平台未列出，尝试最接近的一个或 posix、generic、 c89，按此顺序。

  4. 编译只需片刻， 并在 src 目录中生成三个文件： lua（解释器）、 luac（编译器） 和 liblua.a（库）。

  5. 要检查 Lua 是否正确构建，构建后执行 "make test" 。这将运行解释器并打印其版本。

安装 Lua

一旦你构建了 Lua，你可能想将其安装到系统的官方位置。 在这种情况下，执行 "make install"。官方位置和安装文件的方式在 Makefile 中定义。你可能需要适当的权限来安装文件，因此可能需要执行 "sudo make install"。

要一步构建并安装 Lua，执行 "make all install"， 或 "make xxx install"， 其中 xxx 是你的平台名称。

要在构建后本地安装 Lua，执行 "make local"。 这将创建一个 install 目录，包含子目录 bin、include、lib、man、share， 并按如下方式安装 Lua。 要本地安装 Lua，但在其他目录中，执行 "make install INSTALL_TOP=xxx"，其中 xxx 是你选择的目录。 安装从 src 和 doc 目录开始， 因此如果 INSTALL_TOP 不是绝对路径，请注意。

bin:
lua luac
include:
lua.h luaconf.h lualib.h lauxlib.h lua.hpp
lib:
liblua.a
man/man1:
lua.1 luac.1

这些是你开发所需的唯一目录。 如果你只想运行 Lua 程序， 你只需要 bin 和 man 中的文件。 include 和 lib 中的文件用于 将 Lua 嵌入 C 或 C++ 程序。

自定义

三种事物可以通过编辑文件来定制：

  • 在何处以及如何安装 Lua — 编辑 Makefile。
  • 如何构建 Lua — 编辑 src/Makefile。
  • Lua 功能 — 编辑 src/luaconf.h。

你实际上不需要编辑 Makefile，因为你可以在调用 make 时在命令行设置相关变量。 但是，最好编辑并保存 Makefile 以记录你所做的更改。

另一方面，如果你需要自定义某些 Lua 功能， 请在构建和安装 Lua 之前编辑 src/luaconf.h。 编辑后的文件将是安装的文件， 它将被你构建的任何 Lua 客户端使用，以确保一致性。 专家可以通过编辑 Lua 源代码进行进一步的自定义。

在其他系统上构建 Lua

如果你不使用通常的 Unix 工具，则构建 Lua 的说明取决于你使用的编译器。你需要为构建库、解释器和编译器创建项目（或你的编译器使用的任何内容），如下所示：

库:
lapi.c lcode.c lctype.c ldebug.c ldo.c ldump.c lfunc.c lgc.c llex.c lmem.c lobject.c lopcodes.c lparser.c lstate.c lstring.c ltable.c ltm.c lundump.c lvm.c lzio.c lauxlib.c lbaselib.c lcorolib.c ldblib.c liolib.c lmathlib.c loadlib.c loslib.c lstrlib.c ltablib.c lutf8lib.c linit.c
解释器:
库, lua.c
编译器:
库, luac.c

要在自己的程序中将 Lua 作为库使用，你需要知道如何 使用编译器创建和使用库。此外，要动态加载 Lua 的 C 库，你需要知道如何创建动态库， 并确保这些动态库可以访问 Lua API 函数 — 但不要将 Lua 库链接到每个动态库中。 对于 Unix，我们建议将 Lua 库静态链接到宿主程序中， 并为其符号导出以用于动态链接； src/Makefile 为 Lua 解释器执行此操作。 对于 Windows，我们建议 Lua 库为 DLL。 在所有情况下，编译器 luac 都应静态链接。

如上所述，你可以在构建 Lua 之前编辑 src/luaconf.h 以自定义某些功能。

自 Lua 5.4 以来的变更

以下是 Lua 5.5 中引入的主要变更。 参考手册列出了 不兼容性。

主要变更

  • 全局变量声明
  • for 循环变量为只读
  • 浮点数以足够精度的十进制打印，以便正确读回
  • 构造器更多层级
  • table.create
  • utf8.offset 也返回字符的最终位置
  • 外部字符串（使用 Lua 不管理的内存）
  • 新函数 luaL_openselectedlibs 和 luaL_makeseed
  • 主要垃圾回收增量执行
  • 更紧凑的数组（大数组内存使用减少约 60%）
  • lua.c 动态加载 'readline'
  • 静态（固定）二进制文件（在内存中加载二进制代码块时，Lua 可以在某些内部结构中重用其原始内存）
  • dump 和 undump 重用所有字符串
  • 辅助缓冲区在创建最终字符串时重用缓冲区
